# 이슈 #253 재검토 — 환경별 PDF 렌더링 차이가 Visual Diff 파이프라인에 미치는 영향

## 배경

PR #360 정황에서 발견:
- 동일 HWP (`samples/2022년 국립국어원 업무계획.hwp`)
- 컨트리뷰터 (planet6897) 한컴 PDF: **37 페이지**
- 작업지시자 한컴 PDF: **35 페이지**
- 페이지네이션 자체가 환경별로 다름

이는 한컴 자체의 환경 의존성 (한컴 버전 / 폰트 설치 / OS / 출력 방법 / 한컴 설정 등) 을 시사.

## 이슈 #253 의 핵심 가정

이슈 #253 은 **한컴 PDF 를 권위 기준** 으로 삼는 Visual Diff 파이프라인 구축 제안:

```
[목표 — 한컴 기준]
rhwp SVG (오늘) ↔ 한컴 PDF → 이미지 (고정 기준)   기준점: 한컴
```

핵심 가정: **"한컴 PDF" = 단일 권위 기준** (= 모든 환경에서 결정적, 재현 가능).

## 가정 위반 — 환경별 PDF 차이 발견

본 PR #360 정황은 가정의 핵심을 위반:

| 환경 | 페이지 수 | 시각 결과 |
|------|---------|---------|
| 컨트리뷰터 한컴 | 37 | 페이지 분할 A 패턴 |
| 작업지시자 한컴 | 35 | 페이지 분할 B 패턴 |
| (다른 한컴 환경) | ?? | (검증 필요) |

가능한 원인:
- **한컴 버전**: 한컴2018 / 한컴2020 / 한컴2024 / 한글뷰어
- **폰트 설치 상태**: 함초롬바탕 등 한컴 번들 폰트 가용성
- **OS / 프린터 드라이버**: PDF 출력 경로
- **한컴 설정**: 페이지 분할 옵션, 빈 줄 감추기 등
- **출력 방법**: "PDF로 저장" / "인쇄 → PDF 프린터" / 외부 변환 도구

## 영향 분석 — 이슈 #253 파이프라인 무효화 가능성

### 시나리오 1: 단일 환경 시드 (현재 이슈 #253 의 제안)

`tests/reference_pdf/` 에 작업지시자 한컴 환경의 PDF 만 시드:
- ✅ 작업지시자 환경에서는 결정적
- ❌ 다른 환경의 한컴과는 다를 수 있음 — "한컴 기준" 이라기 보다는 **"작업지시자의 한컴 기준"**
- ❌ 컨트리뷰터가 PR 제출 시 자기 환경의 한컴과 비교 → CI 시드 (작업지시자 한컴) 와 다른 결과 → 컨트리뷰터 혼란
- ❌ rhwp 가 어느 한컴 환경에 맞춰야 하는지 불분명

### 시나리오 2: 다중 환경 시드

`tests/reference_pdf/{한컴2020,한컴2024,...}/` 환경별 시드:
- ⚠️ 어느 환경을 정답으로 삼을 것인가?
- ⚠️ 환경별로 픽셀 diff 결과가 달라 통합 판단 어려움
- ⚠️ 시드 관리 비용 증가 (환경별 동일 샘플 PDF N개 유지)

### 시나리오 3: "한컴 기준" 추상화 — 다중 환경의 공통 부분만

페이지 콘텐츠는 같지만 페이지 분할이 다른 경우:
- 페이지 단위 픽셀 diff 는 무효 (페이지 정렬 자체가 다름)
- 콘텐츠 단위 (paragraph / 표 / 이미지 단위) diff 추상화 필요
- 도입 비용 매우 큼

## 현실 평가

이슈 #253 의 핵심 가정 ("한컴 PDF = 단일 권위 기준") 은 **부분적 무효** 가능성 높음:

1. **rhwp 자체 회귀** (svg_snapshot) 는 여전히 유효 — rhwp 자기 일관성
2. **단일 환경 한컴 PDF 비교** 는 "그 환경의 한컴" 과 일치 여부 — 제한적 가치
3. **다중 환경 한컴 호환** 은 본 PR #360 정황으로 보면 한컴 자체가 비결정적

PR #251 (선례) 의 3건 수정 (PUA 심볼, 문단 border margin, 표 외곽 테두리) 은 **페이지 분할 무관 시각 결함** — 환경 의존성 영향 적음. 그러나 **페이지 분할 결함** (Task #356, #359, #361, #362 등) 은 환경 의존성 직격.
