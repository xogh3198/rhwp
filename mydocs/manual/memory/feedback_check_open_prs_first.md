이슈 작업에 착수하기 전 반드시 열린 PR 목록을 점검한다. 누락 시 외부 기여자가 같은 이슈를 독립적으로 해결한 PR 과 충돌/중복되어 그들의 시간이 헛되게 된다.

**Why:** 2026-04-23 에 이슈 #259 작업 중 @planet6897 이 같은 이슈에 대해 PR #264 를 올렸음을 인지하지 못한 채 내부 브랜치 `local/task259` 에서 독자적으로 해결 → devel 머지 → push. 기여자는 자신의 5-Stage 완주 작업이 중복임을 알고 자진 close. 기여자에게 뼈아픈 경험을 안김. 두 작업의 결론 (HY 7종 매핑 + 영문명 + mod tests) 이 놀랍도록 일치했음에도 내부 작업은 기여자 작업을 전혀 참조하지 못함.

**How to apply:**
1. 이슈 번호를 받고 브랜치 생성 전 **반드시** 다음 실행:
   ```
   gh pr list --repo edwardkim/rhwp --state open --search "in:title,body #<issue>"
   gh issue view <issue> --repo edwardkim/rhwp --json body  # 이슈 본문에 PR 언급 있는지
   ```
2. 관련 PR 발견 시:
   - 기여자 작업을 먼저 읽고 리뷰 → 적합하면 그쪽 경로로 머지 (내부 작업 보류)
   - 부족한 부분이 있으면 코멘트로 피드백 후 기다림
   - 기여자 접근과 내부 계획이 크게 다르면 이슈에 코멘트로 공지 후 협의
3. 오늘할일에 이슈를 착수 항목으로 올릴 때도 "열린 PR 없음 확인" 을 체크리스트에 포함.
4. **절대 금지**: 외부 기여자 PR 의 존재를 모른 채 같은 이슈를 내부 머지하여 해당 PR 을 무의미화하는 행위.
