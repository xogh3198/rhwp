**한컴 PDF 출력은 정답지 역할로 검증되지 않았다.** PDF 가 정답지로서 신뢰할 수 있다는 사실 자체가 입증되기 전까지는 검증 기준으로 수용할 수 없다.

**Why (PR #360 정황):**
동일 HWP (`samples/2022년 국립국어원 업무계획.hwp`) 를 두 환경의 한컴이 출력한 PDF 페이지 수가 다름:
- 컨트리뷰터 (planet6897) 환경: 37 페이지
- 작업지시자 환경: 35 페이지

페이지 분할 자체가 환경 의존적. 따라서 "PDF 와 일치" 검증은 **"누구의 한컴" 인지** 가 미정의 시 무의미.

**잠재 원인 (가설):**
- 한컴 버전 (한컴2018 / 2020 / 2024 / 한글뷰어)
- 폰트 설치 상태 (함초롬바탕 등 한컴 번들 폰트)
- OS / 프린터 드라이버 / PDF 출력 경로
- 한컴 설정 (페이지 분할 옵션, 빈 줄 감추기 등)
- 출력 방법 ("PDF로 저장" / "인쇄 → PDF 프린터" / 외부 변환)

**How to apply:**

1. **PR 검토 시**:
   - 컨트리뷰터의 "PDF 와 일치" 검증 결과를 1차 기준으로 수용 금지
   - 다른 환경 (작업지시자 등) 에서 재검증 필수
   - PR 머지 결정은 시각 결과 + cargo test 회귀 0 + svg_snapshot 등 결정적 검증으로

2. **task 검증 시**:
   - "PDF 와 일치" 를 검증 항목으로 사용 시 **어느 환경의 한컴인지 명시** (commit, 보고서, CHANGELOG)
   - 작업지시자 환경 / 컨트리뷰터 환경 둘 다 시각 점검 권장
   - 가능하면 결정적 기준 (cargo test 회귀, svg_snapshot) 우선

3. **이슈 #253** (한컴 PDF 기준 Visual Diff 파이프라인):
   - **PDF 가 정답지로 검증되기 전까지 보류**
   - 환경 의존성 정량 측정 + 차이 패턴 분석 후 재검토

4. **외부 컨트리뷰터 안내**:
   - CONTRIBUTING.md / README / PR 템플릿에 명시
   - "PDF 와 일치" 만으로 PR 통과 보장 안 됨
   - 검증 기준은 cargo test + 시각 결과 + (작업지시자 환경 재현)

5. **이미 머지된 task 의 재점검**:
   - v0.7.x 의 페이지 분할 정정 task (#347, #356, #359, #361, #362) 가 어느 환경 PDF 기준이었는지 점검
   - 작업지시자 환경에서 회귀 점검
