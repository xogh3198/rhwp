PR 댓글 작성 시 과도한 표현은 자제한다.

**Why:** 컨트리뷰터가 여러 PR 을 동시 진행하는 경우가 많다 (예: planet6897 — Task #347, #353, #370 등 다수). 매번 같은 톤의 과도한 감사 표현 ("정말 감사합니다", "정성스러운 PR") 은 진정성이 떨어지고 형식적으로 보인다.

**How to apply:**
1. 인사말은 짧게 (예: "안녕하세요" 또는 그냥 본문 시작)
2. PR 의 결함 진단 / 변경 / 검증 등 사실 중심으로 기술
3. 처리 결정 (정상 머지 / 흡수 / close) 명확히 명시
4. 감사 표현은 마지막에 짧게 (예: "감사합니다" — "정말" / "🙏" / 이모지 자제)
5. 반복적인 컨트리뷰터에게는 더 사실 중심으로 (관계가 형성되어 매번 인사할 필요 없음)

좋은 예 (간결):
```
한글 세리프 폴백 체인 변경 PR 검토했습니다.

진단 (SVG font-weight 정상 + 환경별 매칭 한계) 정확하고, Nanum Myeongjo / Noto Serif CJK KR
추가의 영향 범위 평가도 적절합니다. cherry-pick 방식으로 머지했습니다.

검증: cargo test 1014 passed, svg_snapshot 6/6, 샘플 회귀 0, form-002 폰트 체인 적용 확인.

상세: mydocs/pr/pr_371_report.md
감사합니다.
```

피하는 표현:
- "정말 감사합니다"
- "정성스러운 PR"
- "큰 도움이 됐습니다"
- "다음 사이클에서도 좋은 PR 부탁드립니다"
- 🙏 등 감사 이모지 남발
