**v0.7.6 사이클의 페이지 레이아웃 회귀 다수의 진짜 origin 가설**: 외부 컨트리뷰터들이 자기 환경의 PDF 를 정답지로 사용해 PR 검증 → 작업지시자 환경 (한컴 2010 + 2022 편집기) 과 다른 페이지 분할에 맞춰 머지 → 작업지시자 환경에서 회귀로 보임.

**Why (PR #360 정황으로 명시화):**
- 컨트리뷰터 (planet6897) 환경: 한글 Viewer 1.31.7 + macOS Sequoia 인쇄 다이얼로그 → 37 페이지
- 작업지시자 정답지: 한컴 2010 + 한컴 2022 (편집기) → 35 페이지
- 페이지 분할 자체가 환경별로 다름

**v0.7.6 → v0.7.7 정정 패턴:**
| v0.7.6 머지 | v0.7.7 정정 task |
|---|---|
| PR #341 (Task #340), #343 (Task #321~#332), #351 (Task #347) | Task #359 (fit drift), #361 (page_num), #362 (8항목 누적) |

작업지시자가 v0.7.7 에서 정정한 회귀 결함이 **컨트리뷰터 환경에서는 결함으로 보이지 않았을 가능성** — 다른 페이지 분할이라 회귀 미인지.

**How to apply:**

1. **외부 페이지 레이아웃 PR 검토 시**:
   - 컨트리뷰터의 PDF 검증 환경 확인 (한글뷰어 vs 한컴 편집기)
   - 한글뷰어 / macOS 인쇄 / 외부 변환 도구 출력은 정답지 아님 → 머지 보류 + 작업지시자 환경 재검증 필수
   - "작업지시자가 직접 시각 검증" 단계를 PR 머지 전 필수 게이트로

2. **컨트리뷰터 안내**:
   - PR 본문에 PDF 생성 환경 명시 의무 (한글 도구 + 버전 + OS + 출력 방법)
   - 한컴 2010 또는 2022 편집기가 아니면 그대로 통과 보장 안 됨 명시
   - 한컴 편집기 부재 시 시각 비교 결과는 참고만 (필수 검증 아님)

3. **머지 후 회귀 추적**:
   - v0.7.x 의 페이지 분할 정정 task 들이 외부 PR 의 환경 차이로 발생한 회귀일 가능성 의심
   - 일반화 알고리즘 흡수 시 (PR #374 사례) 회귀 위험 강함

4. **장기 대응**:
   - 작업지시자 정답지 환경 자동화 (한컴 2010/2022 PDF 시드 가능 시)
   - 그 전까지는 메인테이너 직접 시각 검증을 머지 게이트로
