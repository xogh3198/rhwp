페이지 총 수 / SVG byte 비교만으로는 셀 안 그림 크기 결함, 미세 좌표 어긋남 같은 시각 결함을 검출하지 못한다. 같은 페이지 수 + 같은 byte 가 나와도 셀 경계 초과, 인접 컨텐츠 겹침 같은 시각상 결함은 통과한다.

**Why:** 컨트리뷰터 9명+ 활발한 사이클에서 자동 검증 게이트만 통과한 PR 들이 누적되면서, 작업지시자 시각 판정에서 새 결함 정황이 발견되는 빈도가 높아짐 (Task #474 정정 직후 작업지시자가 16쪽 두 번째 표 셀 안 그림 확대 결함 #477 발견). PR 처리 흐름에서 자동 검증만으로는 부족함이 드러남.

**How to apply:**
- PR 머지/체리픽 결정 시 `cargo test + svg_snapshot + clippy + WASM 빌드 byte 수 + 페이지 총 수 byte 비교` 4 게이트 만으로 안심하지 말 것
- 작업지시자 직접 시각 판정을 1차 권한 게이트로 인지 (메모리 `feedback_v076_regression_origin` 정합)
- 시각 회귀 테스트가 PR 사이클이 늘수록 비중이 커진다 — PR 처리 종료 후 누적 통합 시각 검증 단계가 절차로 자리잡아야 함
- 발견된 시각 결함은 별도 이슈로 등록 (PR 의 본질 정정과 분리) — Task #477 처럼
- 향후: 시각 회귀 자동화 (이미지 픽셀 diff / 셀 안 그림 박스 검증) 같은 도구 후보
