# Task #370 수행계획서: 한글 세리프 폴백 체인에 Noto Serif CJK KR 추가

## 1. 목적

`form-002.hwpx` 10쪽 등에서 볼드로 지정된 한글 문구가 SVG 렌더링 시 볼드로 표시되지 않는 문제를 해소한다. SVG 출력 자체에는 `font-weight="bold"`가 정확히 부여되어 있으나, 폴백 체인에 등록된 모든 한글 세리프 패밀리(`함초롬바탕`, `Batang`, `바탕`, `AppleMyungjo`, `Noto Serif KR`)가 표준 리눅스 환경에 미설치되어 generic `serif`로 떨어지고, 일부 렌더러에서 한글 자동 폴백 글리프에 bold variant가 매칭되지 않는 것이 원인이다.

## 2. 현황

- 폴백 체인 정의: `src/renderer/mod.rs:558` 및 `:565`
- 현재 체인: `'Batang','바탕','AppleMyungjo','Noto Serif KR',serif`
- 리눅스 시스템에 설치된 한글 세리프: `Noto Serif CJK KR` (Regular/Bold 포함). 체인에 누락.
- Sans-serif 체인(`mod.rs:542,568`)은 `Noto Sans KR`만 포함 — 동일한 위험. 본 타스크에서는 우선 세리프만 다룬다(이슈 범위).

## 3. 변경 범위

| 파일 | 변경 내용 |
|------|----------|
| `src/renderer/mod.rs` | 세리프 폴백 체인 두 곳(`:558`, `:565`)에 `'Noto Serif CJK KR'` 추가 |
| `src/renderer/mod.rs` (테스트) | `test_generic_fallback` 의 기대 문자열 갱신(`:935` 부근) |

체인 위치: `'Noto Serif KR'` 직후, `serif` 직전. (Google Fonts 단독 패밀리 → 시스템 CJK 패밀리 → generic 순서)

## 4. 검증

1. `cargo build` 통과
2. `cargo test` 통과 (폴백 테스트 포함)
3. `rhwp export-svg samples/hwpx/form-002.hwpx -p 9` 출력 SVG에서 `Noto Serif CJK KR` 포함 확인
4. 출력 SVG를 브라우저에서 열어 (기술적 측면), 연구개발기간 등 볼드 문구 시각 확인

## 5. 작업 단계 (3단계)

- **Stage 1**: 세리프 폴백 체인 수정 + 테스트 갱신 + `cargo test`
- **Stage 2**: form-002.hwpx 10쪽 SVG 재출력 + 시각 검증
- **Stage 3**: 단계 보고서 정리, 최종 보고서 작성, merge 준비

## 6. 영향 범위

- HTML 렌더러도 `generic_fallback`을 공유하므로 동일하게 개선됨 (`html.rs:267`).
- PDF 렌더러는 `add_font_fallbacks` 별도 처리(`pdf.rs:25`) — 본 변경 영향 없음.
- 기존 환경(Windows/macOS)에서 우선순위 변경 없음 — `Noto Serif CJK KR`은 체인 후순위에 추가하므로 기존 폰트 매칭 결과 유지.
