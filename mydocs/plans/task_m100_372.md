# Task #372 — SVG 렌더러 형광펜 배경(charPr.shadeColor) 누락 (간소 계획서)

## 배경

`samples/hwpx/hwpx-h-02.hwpx` 9쪽에 PDF 정답에서는 다수 단어가 민트색(`#CDF2E4`) 배경으로 강조되어 있으나, `rhwp export-svg` 출력에는 형광펜 배경이 전혀 없다. HWPX `<hh:charPr ... shadeColor="...">` 5건이 IR/`TextRunStyle.shade_color`까지는 정상 보존되며, SVG 렌더러의 `TextRun` 경로만 이 필드를 무시하는 것이 직접 원인.

대조 (다른 렌더러는 정상 동작):
- `src/renderer/html.rs:334-337` — `background-color` CSS
- `src/renderer/web_canvas.rs:1271-1279` — `fill_rect` (텍스트 폭 × `font_size * 1.2`, 위치 `y - font_size`)

## 변경 범위

`src/renderer/svg.rs` 단일 파일.

`RenderNodeType::TextRun` 처리에서 일반(회전 없음, char_overlap 없음) 경로 진입 직전(약 240행 부근)에 다음 처리를 추가:

1. `style.shade_color` 의 RGB 부분이 흰색(`0xFFFFFF`) 또는 0이 아닐 때만 동작.
2. 텍스트 폭은 web_canvas와 동일하게 `compute_char_positions`(또는 `node.bbox.width`)에서 얻는다. SVG는 이미 `node.bbox`로 그리므로 `node.bbox.x/width`를 사용해도 충분하며 web_canvas와 시맨틱 일치.
3. 위치/크기: web_canvas와 동일하게 `y - font_size` 시작, 높이 `font_size * 1.2`.
4. `<rect>` 출력 (텍스트 `<text>` **앞**에 그려야 텍스트가 위에 보임).

회전 텍스트 / `char_overlap` 경로는 기존 동작 유지 (web_canvas도 일반 경로에만 형광펜을 적용).

## 검증

1. `cargo build --release` 성공.
2. `rhwp export-svg samples/hwpx/hwpx-h-02.hwpx -p 8 -o output/svg/h02-p9/` 재출력.
3. 결과 SVG에 `#CDF2E4` 또는 `#cdf2e4` `<rect>` 등장 확인.
4. PDF 9쪽과 시각 비교: "거주자, 외국법인, 증권을 취득, 금전 대여, 주식 또는 출자지분, 추가취득, 외국법인에 투자한 거주자, 상환기간 1년 이상의 금전 대여, 영업소, 설치·확장·운영, 해외사업 활동, 자금을 지급" 12개 강조구 모두 민트 배경 표시.
5. `cargo test` 회귀 없음.

## 위험/비범위

- 본 수정은 SVG 렌더러에만 영향. HTML/Canvas/Layout 변경 없음.
- 음영색이 `none`인 기존 문서는 RGB 가드(`!= 0xFFFFFF && != 0`)로 영향 없음.
- 표/머리말/꼬리말의 음영색도 동일 경로를 타므로 자동 적용됨.
