# 수행 계획서 — Task M100 #394

## 이슈

[#394](https://github.com/edwardkim/rhwp/issues/394) — rhwp-studio 표 셀 진입 시 투명선 자동 ON 로직 제거 (기본값 OFF 유지)

## 목적

rhwp-studio 의 표 편집 UX 를 한컴 뷰어 출력과 시각 일치시킴. 투명선은 사용자가 명시 토글한 경우에만 표시.

## 현재 동작 (수정 전)

| 상태 | 동작 |
|------|------|
| 문서 로드 직후 | 투명선 OFF (WASM 코어 기본값) |
| 표 셀 안 커서 진입 | **자동 ON** (`checkTransparentBordersTransition`) |
| 표 셀 밖 커서 탈출 | 자동 OFF (auto 였던 경우) |
| 사용자 토글 (`Alt+V → T` / 메뉴) | manual 상태로 전환 — 수동 ON/OFF |

## 변경 후 동작

| 상태 | 동작 |
|------|------|
| 문서 로드 직후 | 투명선 OFF |
| 표 셀 안/밖 커서 이동 | **자동 토글 안 함** — OFF 유지 |
| 사용자 토글 (`Alt+V → T` / 메뉴) | 수동 ON/OFF — 기존과 동일 |

## 변경 범위 — 향후 되돌리기 가능한 주석 처리 방식

**rhwp-studio 한 파일**:

| 파일 | 변경 |
|------|------|
| `rhwp-studio/src/engine/input-handler.ts` | `checkTransparentBordersTransition()` 호출 지점 (line 1503, 1520) 을 **주석 처리** + 메서드 본체 / 관련 상태 변수 (`autoTransparentBorders`, `manualTransparentBorders`, `wasInCell`) 도 **주석 처리로 보존** |

**주석 처리 방침** (작업지시자 지시 반영):

- **삭제 금지** — 향후 정책이 다시 바뀔 가능성 (예: 사용자 피드백에 따라 자동 ON 복귀) 에 대비
- 호출 지점 주석에 **`[Task #394]` 마커 + 사유 + 되돌리기 방법** 명시
- 메서드 본체도 보존하되 호출되지 않으므로 dead code 경고 회피를 위해 한 줄 사용 유지 (예: 더미 reference) 또는 `@ts-ignore` 코멘트 활용
- 제거 시점이 도래하면 별도 task 로 깔끔히 정리

예시:
```typescript
// updateCaretAndScroll() 내부
this.updateSelection();
this.emitCursorFormatState();
// [Task #394] 셀 진입 자동 ON 로직 비활성화 — 한컴 출력 정합성을 위해 OFF 기본값 유지.
// 되돌리려면 아래 호출의 주석을 풀고 상태 변수 / 메서드 본체의 주석도 동시에 해제.
// this.checkTransparentBordersTransition();
this.updateFieldMarkers();
```

`manualTransparentBorders` 는 `transparent-borders-changed` 이벤트 수신 (line 391-393) 에서도 사용되지만, 자동 로직이 멈추면 manual 추적이 불필요 → 같이 주석 처리. 단 사용자 토글 자체는 `view.ts` 에서 wasm 상태로 직접 읽어 동작하므로 영향 없음.

**변경 없음**:
- WASM 코어 (`src/document_core/`, `src/renderer/layout.rs`, `src/wasm_api.rs`)
- 단축키 / 메뉴 토글 (`Alt+V → T`, `view:border-transparent`)
- 기존 토글 명령어 (`view.ts:110-124`)
- 다른 확장 (rhwp-chrome, rhwp-firefox, rhwp-vscode)

## 검증 방법

### 자동 검증

- `cargo test --lib` (회귀 없음 — WASM 변경 없음)
- `cargo test --test svg_snapshot` (회귀 없음 — 렌더 변경 없음)
- `npm run build` (rhwp-studio TypeScript 빌드 통과)
- E2E (`text-flow.test.mjs`) 회귀 없음

### 시각 / 수동 검증

- 문서 로드 → 투명선 OFF 상태 확인
- 표 셀 클릭 / 키보드 이동 → 투명선 자동 ON 안 일어남 확인
- 표 셀 밖 이동 → 변화 없음 확인
- `Alt+V → T` → 투명선 수동 ON 확인 / 다시 누르면 OFF
- 메뉴 [보기] → [투명 선] 토글 동작 확인
- 셀에서 표 외부로 커서 이동 시 manual ON 상태가 유지되는지 확인

## 단계 분할 (구현 계획서 후속)

3 단계 예상:

1. **Stage 1**: `checkTransparentBordersTransition()` 호출 지점 (line 1503, 1520) 주석 처리 + `[Task #394]` 마커 / 되돌리기 방법 코멘트
2. **Stage 2**: 메서드 본체 + 관련 상태 변수 (`autoTransparentBorders`, `manualTransparentBorders`, `wasInCell`) + `transparent-borders-changed` 핸들러 주석 처리. dead code 경고 회피 (TS unused 경고 처리)
3. **Stage 3**: 빌드 / 회귀 테스트 + 시각 검증 + WASM 빌드 확인

상세 단계는 구현 계획서에서 확정.

## 위험 / 주의

- 자동 ON 의 사용성 (셀 경계 가이드) 가 사라지므로, **사용자가 토글 단축키를 모를 경우 표 편집이 어려워질 가능성** — 단축키는 메뉴 / 도구 상자에 노출되어 있어 발견 가능
- `Initial commit (v0.5.0)` 부터 있던 디자인 결정의 의도적 reversal — 이슈 본문에 사유 명시 완료
- E2E 테스트가 셀 진입 / 탈출 시점의 자동 토글에 의존하지 않는지 점검 필요 (대부분 텍스트 플로우 검증이라 의존 가능성 낮음)
