# Task #398: 표 페이지 분할 시 rowspan>1 셀 누락 — 수행계획서

## 목표

표 페이지 분할 알고리즘이 rowspan>1 셀의 시각적 점유 영역을 분할 단위에 반영하도록 개선하여 PDF/한글과 동일한 페이지 분할 동작을 달성한다.

## 배경

- **재현 문서**: `samples/2025년 기부·답례품 실적 지자체 보고서_양식.hwpx`
- **증상**: 1쪽 하단에 표(pi=22, 3×3) 첫 행이 잘려 들어가고 제목 셀(rowspan=2) 텍스트 "<2025년 기부·답례품 실적 분석보고서 요약>" 가 본문 영역 끝에 겹쳐 그려짐. PDF는 표 전체를 2쪽으로 미룸.
- **분석 결과**:
  - 페이지 1 잔여 공간 = 19px
  - `mt.row_heights[0]` = 17.27px (rs=1 셀만 집계)
  - 실제 첫 분할 단위 = 행 0+1 묶음(rowspan=2 셀 점유) = 35.4px
  - 19 ≥ 17.27 → 행 0 배치 결정 → 35.4px 셀이 잘림

## 수정 범위

### 핵심 변경

1. **`MeasuredTable::row_block_for(row) -> (start, end, height)`** 신규 추가
   - 입력 행에서 시작하여, 그 행을 포함하는 모든 rowspan 셀의 최대 종료 행까지를 한 블록으로 묶어 반환
   - 블록 높이 = `range_height(start, end)` 활용

2. **`split_table_rows`의 분할 판정에 블록 적용** (`pagination/engine.rs:1492~1507`)
   - `first_row_h` 대신 `first_block_h = row_block_for(0).2` 사용
   - 잔여 공간 < 블록 높이 → 다음 페이지로 advance

3. **`find_break_row` 결과를 블록 경계로 스냅** (`height_measurer.rs:1179`)
   - 후보 종료 행이 rowspan 묶음 중간이면 묶음 시작 행으로 후퇴
   - 강제 분할 경로(인트라-로우 split)는 영향 없음 (이미 단일 행 처리)

### 부가 변경

- `is_row_splittable` / `min_first_line_height_for_row` 는 이번 수정 범위에서 제외 (행 단위 검사이므로 블록 정책과 별개로 동작)
- 단, 새 헬퍼는 향후 셀 단위 분할 정책 확장 시 재사용 가능하도록 설계
