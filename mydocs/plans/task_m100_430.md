**이슈**: [#430 그림 자동 크롭(박스 비율 맞춤) 미구현 — exam_kor 헤더 "국어 영역(A 형)"](https://github.com/edwardkim/rhwp/issues/430)

## 1. 문제 정의

`samples/exam_kor.hwp` 1쪽 상단 가운데 헤더(국어 영역 제목 이미지)가 SVG 출력에서는 `국어 영역(A 형)`으로 보이지만, 한컴이 만든 PDF(samples/exam_kor.pdf)는 `국어 영역`만 보인다. PDF가 정답.

같은 이미지(BinData bin_id=27)가 섹션 0(국어), 1(화법과 작문), 2(언어와 매체) 각각의 본문 첫 문단 1×1 표 안의 그림 컨트롤에 사용된다.

## 2. 원인 가설 (분석 단계에서 확인된 사실)

| 항목 | 값 |
|------|-----|
| HWP 내부 BinData 원본 JPEG | 2320 × 354 px (가로/세로 6.55) |
| 실제 박힌 텍스트 | `국어 영역(A 형)` |
| HWP 그림 컨트롤 표시 박스 | 57.8 × 15.0 mm (가로/세로 3.85) |
| 한컴 PDF가 페이지에 끼운 비트맵 | 1137 × 295 px (가로/세로 3.85) — `국어 영역`만 |

원본 비율(6.55) ≠ 박스 비율(3.85). 한컴은 박스 비율에 맞춰 우측을 잘라낸 비트맵을 생성해 PDF에 박았다. rhwp는 `preserveAspectRatio="none"`으로 박스에 그대로 늘여 채우므로 `(A 형)` 부분까지 모두 보이고 가로 방향이 압축돼 보인다.

두 가지 시나리오가 가능하므로 1단계에서 확정한다:

- **(A) HWP에 명시 crop이 있다** — `pic.crop`(left/top/right/bottom)에 자르기 좌표가 들어있으나 SVG 렌더러가 잘못 처리.
- **(B) HWP에 명시 crop이 없다** — 한컴 자체가 *박스 비율에 맞춰 자동 클리핑*하는 동작을 한다. rhwp도 같은 자동 클리핑을 추가해야 함.

직관적으로 (B) 가능성이 높다(이미지/박스 비율 불일치 시 한컴은 축소가 아닌 잘라내기 동작). 1단계에서 실측으로 확정.

## 3. 영향 범위

### 3.1 직접 수정 대상 (예상)

- `src/renderer/svg.rs::render_image_node` (1086-1141 부근) — `FitToSize` 분기에서 자동 클리핑 처리 추가.
- `src/renderer/web_canvas.rs::render_image_node` (2112-2130 부근) — 동일 처리.
- `src/renderer/layout.rs:2606-2621` — `crop` 전달 부분(데이터 변환 필요 시).

### 3.2 비수정/조사 대상

- `src/parser/control/shape.rs:833-837` — `pic.crop` 파싱 (실측 후 결정).
- `src/main.rs::dump` — 표 셀 내부 그림 컨트롤의 crop 출력 누락은 보강 가능(별도 작업으로 분리 가능).

## 4. 수행 절차 (4단계)

### Stage 1: 원인 정밀 조사 + 수정 방침 확정

- bin_id=27 그림 컨트롤의 실제 `pic.crop` 값을 확인한다. 수단:
  - `src/main.rs:1643` 표 셀 내부 그림 dump 형식에 `crop=(...)` 추가(임시/유지 둘 다 가능).
  - 또는 일회용 통합 테스트로 직접 IR을 읽어 확인.
- 결과를 보고 시나리오 A/B를 확정 → 구현 계획서 작성.
- 산출물: `mydocs/plans/task_m100_430_impl.md`, `mydocs/working/task_m100_430_stage1.md`.

### Stage 2: SVG 렌더러 수정

- 확정된 방침대로 `src/renderer/svg.rs::render_image_node` 수정.
  - 시나리오 (B)인 경우: `crop = None && fill_mode == FitToSize` 일 때 원본 이미지 비율과 박스 비율을 비교해, 다르면 src 영역을 좌측·상단 정렬로 자동 클리핑(중첩 `<svg viewBox=...>` 사용 — 기존 crop 분기 코드 재활용).
  - 시나리오 (A)인 경우: crop 좌표를 올바른 단위로 해석해 적용.
- 단위 테스트 추가(가능 시) 또는 통합 검증으로 대체.
- 산출물: `mydocs/working/task_m100_430_stage2.md`.

### Stage 3: web_canvas 동기화 + 회귀 검증

- `src/renderer/web_canvas.rs::render_image_node`(2112 부근) 동일 로직 적용.
- 회귀 검증:
  - `samples/exam_kor.hwp` 1/14/20쪽 SVG 출력 → 헤더 `국어 영역` 정상, 14쪽은 `(화법과 작문)` 텍스트 별도 표시 정상 유지.
  - `cargo test` 전체 통과.
  - 그림 포함 다른 샘플(`re/`, `samples/` 일부) 시각 회귀 없는지 SVG 비교.
- 산출물: `mydocs/working/task_m100_430_stage3.md`.

## 6. 위험 요소

- 자동 클리핑을 잘못 적용하면, 의도적으로 비율을 무시해 늘이거나 짜부라뜨린 이미지가 변형돼 보일 수 있음 → 시나리오 결정 후 가급적 좁은 범위(헤더형 와이드 이미지 케이스)부터 적용 또는 픽셀 단위 임계로 무시 영역 설정.
- `pic.crop`가 비어있는지 판단 기준(모두 0인 경우 vs 명시된 0)이 모호할 수 있음 → 1단계에서 데이터로 확정.
