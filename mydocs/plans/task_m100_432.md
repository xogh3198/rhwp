## 1. 배경

rhwp-firefox 테스트 중 GitHub wiki 페이지의 HWP 링크를 rhwp hover 카드에서 바로 열면 viewer가 `파일 로드 실패: 유효하지 않은 파일: CFB 오류`를 표시했다.

문제 링크는 `https://github.com/edwardkim/rhwp/blob/devel/saved/pr360-edward.hwp` 이다. 경로는 `.hwp`로 끝나지만 실제 응답은 HWP 바이트가 아니라 GitHub HTML 문서였다.

확인 결과:

- GitHub `blob` URL 응답: `text/html; charset=utf-8`
- GitHub `blob` URL 앞 바이트: `0a 0a 0a 0a 0a 0a 3c 21 ...` (`<!DOCTYPE html`)
- raw URL 앞 바이트: `d0 cf 11 e0 a1 b1 1a e1 ...` (HWP CFB 매직)
- 로컬 `saved/pr360-edward.hwp`: 정상 HWP 5.x 파일

## 2. 일반 문제 정의

이번 문제는 GitHub 링크 하나를 rule base로 치환하는 문제가 아니다. 현재 확장이 아래 두 조건을 사실상 같은 의미로 취급하는 구조가 핵심 원인이다.

1. URL 또는 파일명이 `.hwp`/`.hwpx`로 끝난다.
2. fetch 결과가 실제 HWP/HWPX 문서 바이트다.

웹에서는 `.hwp` 경로가 실제 파일 대신 HTML 미리보기 페이지, 인증 페이지, 오류 페이지, 다운로드 중간 페이지를 반환할 수 있다. viewer가 응답 바이트 검증 없이 곧바로 WASM 파서에 전달하면 사용자는 원인을 알기 어려운 CFB 오류를 보게 된다.

## 3. 목표

### 3.1 1차 목표

브라우저 확장의 HWP/HWPX URL 처리 흐름에 실제 응답 검증 계층을 도입하여 HTML/오류 페이지를 HWP 파서에 넘기지 않도록 한다.

### 3.2 2차 목표

GitHub `blob`처럼 "파일 상세 페이지 URL"이 "실제 파일 바이트 URL"과 다른 경우를 provider adapter 구조로 정규화한다. 첫 적용 대상은 GitHub이지만 설계는 GitLab/Bitbucket/공공 다운로드 페이지로 확장 가능한 형태를 지향한다.

### 3.3 성공 기준

- GitHub wiki의 `blob` HWP 링크 카드 클릭 시 viewer가 실제 raw HWP를 로드한다.
- GitHub blob 페이지의 `View raw`/raw 링크 동작은 유지된다.
- HTML 응답을 HWP 파서에 넘기지 않고 명확한 사용자 오류를 표시한다.
- viewer 로드와 썸네일 추출이 동일한 URL 해석/검증 규칙을 사용한다.
- GitHub 전용 조건문이 호출부 여러 곳에 흩어지지 않는다.

## 4. 통합 전략 원칙

1. **오탐 방어 우선** — 확장자 감지는 후보 판정일 뿐, 실제 파싱 전에는 응답 바이트를 검증한다.
2. **provider adapter 구조** — GitHub 처리는 첫 adapter로 둔다. 호출부에는 provider별 if문을 반복하지 않는다.
3. **경로 통합** — viewer 로드, background fetch, 썸네일 추출이 같은 helper를 사용하도록 정리한다.
4. **보안 감사 준수** — fetch-file 오픈 프록시, sender 검증, HTTP/내부 IP 차단 정책을 약화하지 않는다.
5. **회귀 최소화** — 기존 raw URL, 직접 파일 URL, 로컬 파일 열기 흐름은 유지한다.
6. **브라우저별 차이 확인** — Firefox를 직접 재현 대상으로 하되 Chrome 공통 코드와 Safari 영향도 확인한다.

## 7. 위험 요소

| 위험 | 완화 |
|---|---|
| GitHub adapter가 지나치게 지엽적인 rule이 됨 | provider adapter 인터페이스로 격리하고 호출부는 공통 helper만 사용 |
| HTML 응답을 너무 늦게 감지해 기존 CFB 오류가 반복됨 | WASM 파서 호출 직전 매직 검증을 필수 게이트로 둠 |

## 8. 검증 방법

- 순수 함수 테스트:
  - GitHub `blob` HWP URL -> raw URL 변환
  - raw URL은 변환하지 않음
  - 일반 HWP URL은 변환하지 않음
  - HWP CFB 매직 감지
  - HWPX ZIP 매직 감지
  - HTML 응답 감지
- 수동 검증:
  - GitHub wiki 카드 클릭 -> viewer 로드 성공
  - GitHub blob 페이지 raw 링크 -> viewer 로드 성공
  - HTML 응답 URL -> 명확한 오류 표시
  - hover 썸네일 표시 회귀 확인
