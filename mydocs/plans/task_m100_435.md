## 배경

`samples/exam_kor.hwp` (2025학년도 수능 국어 문제지) 가 한컴 PDF 출력은 **20 페이지** 인 반면 rhwp 는 **24 페이지** 로 출력. 마지막 페이지 푸터가 `24/20` 으로 표시되며, PDF 와 비교 시 거의 빈 페이지 (page 2: 49px, page 15: 104px) 가 삽입되고 일부 페이지의 컬럼 활용이 부족함.

## 현상 요약 (조사 단계 결과 기반)

`rhwp dump-pages` 진단 결과 3 가지 메커니즘으로 분리됨:

### (1) 컬럼 over-fill → orphan + hard break (page 2, 15)

| 위치 | rhwp used | hwp_used | diff |
|---|---|---|---|
| page 1 단 1 (pi=0.30 split) | 1207.6 | 1153.1 | **+54.5px** |
| page 14 단 1 (pi=1.25 split) | 1193.5 | 1087.4 | **+106.2px** |

직후 paragraph 가 `[쪽나누기]` → `force_new_page()` 가 미사용 컬럼을 비운 채 진행 → 거의 빈 page 2 (49px), page 15 (104px) 발생.

→ **#393 의 (A) col 1 reserve 과대 + (B) 항목별 누적 과대** 효과.

### (2) 일반 페이지 누적 -100~-300px (page 4, 5, 7, 8, 10, 12, 13, 19)

표/도형 (Shape) 배치 후 컬럼 잔여 공간 산정이 한컴 대비 작게 산정되어 다음 paragraph 가 일찍 다음 컬럼/페이지로 밀림.

| 페이지 | 단 0 diff | 단 1 diff |
|---|---|---|
| 4 | -250.1 | -212.1 |
| 5 |  -42.8 | -333.6 |
| 12 | -268.5 | -276.7 |
| 13 | -329.0 | -130.3 |

### (3) Square wrap 표 + 컬럼 불균형 (page 16)

| 위치 | rhwp used | hwp_used | diff |
|---|---|---|---|
| page 16 단 0 | 1225.8 (본문 1211.3 초과) | — | overflow 14px |
| page 16 단 1 | 64.3 | 1189.0 | -1124.7 |

`paginate_text_lines` (`engine.rs:702-711`) 의 표 직후 trailing `line_spacing` 제외 로직이 단 0 over-fill 을 통과시킴.

각 우선순위 단계 후 `rhwp export-svg` 로 페이지 수를 확인하고 기존 회귀 케이스 (exam_eng 8쪽, k-water-rfp 27쪽 등) 를 검증한 후 다음 단계로.

```
[현 상태] 24 페이지
    ↓ 1단계 (#393 옵션 A)

    ↓ 2단계 (Shape/Table 잔여 공간)
[목표] 20 페이지 (누적 -100~-300 해소)
```

### 진단 도구 활용

- `rhwp dump-pages -p N` — 페이지별 컬럼 사용 / hwp_used 차이
- `rhwp dump -s S -p P` — ParaShape, LineSeg, 표 속성 상세
- `rhwp ir-diff sample.hwpx sample.hwp` — HWPX/HWP IR 차이 (가능 시)
- `RHWP_TYPESET_DRIFT=1` 환경변수 — typeset cur_h vs HWP vpos 추적 (#393 에서 사용된 진단)

## 위험 요소

1. **다단 reserve 로직은 `exam_eng.hwp` (8쪽), `k-water-rfp.hwp` (27쪽) 등 회귀 가능성 있음** — Task #391, #393, #386 의 누적 결과를 깨지 않도록 단계별 회귀 검증 필수
2. **`pi=0.30` 의 인라인 그림 (체크 표시) 영향** — line 0 의 lh=1491 이 image 영향. 보정 적용 시 다른 문서의 인라인 이미지 케이스 영향 조사 필요
3. **(2) 의 누적 부족이 (1) 의 over-fill 과 정반대 방향이라는 점** — 단순한 단방향 보정으로 해결 안 됨. 표/도형 배치 후 잔여 공간 계산 로직과 일반 paragraph 누적 로직이 별도 경로일 가능성
