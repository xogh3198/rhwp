## 1. 배경

Task #435 에서 exam_kor.hwp 가 24→22 페이지로 단축되었으나, HWP 원본은 21 페이지이다.
22→21 페이지 차이의 원인은 페이지 14 col 0 의 over-fill (1225.8 > 1211.3 px, 14px 초과) 로 추정된다.

페이지 14 col 0 에는 Square wrap 표 4 개 (pi=33, pi=37, pi=40, pi=47) 가 포함되며,
표 직후 paragraph fit 검사에서 trailing line_spacing 을 빼는 로직 (engine.rs:702-707) 이
14px 초과를 허용 → col 0 over-fill → col 1 거의 비어있는 상태로 페이지 종료
→ 같은 콘텐츠가 page 14 + page 15 의 2 페이지로 분리되는 것으로 보인다.

## 2. 목표

1. 페이지 14 col 0 used ≤ 본문 높이 1211.3 px (현 1225.8 px)
2. 페이지 14 col 1 used ≈ 1189 px (HWP 와 동일 수준)
3. exam_kor.hwp 22 → 21 페이지 (페이지 14 + 15 통합)
4. **회귀 없음**: hwpspec.hwp, basic.hwp 등 회귀 테스트 페이지 수 유지

## 3. 작업 범위

### 3.1 조사 대상

- `src/renderer/pagination/engine.rs:617-637` — 문단 적합성 검사의 `effective_trailing` 로직
- `src/renderer/pagination/engine.rs:697-711` — 표 직후 trailing 제외 분기 (`prev_is_table`)
- Square wrap 표가 col 0 cur_h 누적에 미치는 영향 (table_wrap, typeset 등)
- pi=33, pi=37, pi=40, pi=47 의 ParaShape, line_segs, 표 metric 검증

### 3.2 가설

- **가설 A (강)**: `prev_is_table` 분기에서 trailing_ls 제외가 Square wrap 케이스에는 부적절. Square wrap 표는 본문 흐름에서 분리된 floating box 이므로 직전 표가 아닌 직전 paragraph 의 line_spacing 을 검사해야 함.
- **가설 B**: typeset 단계에서 Square wrap 표 영역 (표 + 좌/우 여백) 의 cur_h 누적이 부정확해 페이지네이션 단계 line_heights 와 어긋남.
- **가설 C**: pi=47 의 line_height 가 부정확 (line_segs 에 표 wrap 영역 누락 또는 중복).

### 3.3 비범위 (Out of Scope)

- 표 wrap 알고리즘 자체의 재설계
- 다른 wrap 모드 (TopAndBottom, BehindText 등) 의 회귀 수정
- typeset 단계 측정 로직 전면 개편
