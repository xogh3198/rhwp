**브랜치**: `local/task445` (분기점: `local/devel`)

## 1. 배경

`exam_kor.hwp` 의 SVG 출력에서 지문 박스(문단 테두리)의 세로선이 PDF 대비 길게 그려진다. 페이지에 따라 길이가 다르며, 페이지 8·15 의 경우 페이지 자체(1587px)를 초과하여 꼬리말/인쇄영역 바깥까지 그려진다.

## 2. 증상 요약

- 영향 범위: `border_fill_id != 0` 인 문단이 페이지 분할(PartialParagraph)되는 모든 문서
- 확인된 페이지: exam_kor.hwp 의 page 2, 5, 8, 15 (총 20페이지 중 4페이지)
- page 8: 세로선이 y=1671 까지 그려짐 (페이지 바닥 1587 + 84px)

## 3. 원인 가설 (1차)

`src/renderer/layout.rs:442-474` 의 `expand_clip()` 는 자식 bbox 가 `body_area` 를 초과하면 body clip rect 을 확장만 하고 축소하지 않는다. 표 외곽 테두리 누락 방지 목적이지만, 분할 문단의 bbox 가 col_bottom 을 초과해 들어오면 clip 이 페이지를 넘어선 영역까지 확장되고, 그 안의 paragraph border(세로선)가 그대로 그려진다.

**근본 원인 후보**:
- (A) PartialParagraph 의 bbox 자체가 잘못 계산됨 (분할 전 전체 문단 높이가 들어감)
- (B) bbox 는 정상이지만 paragraph border 그리기 단계에서 분할 라인 수가 아닌 전체 문단 높이로 그림
- (C) expand_clip 의 정책이 부적절 (col_bottom 으로 제한해야 함)

## 4. 작업 범위

- **포함**:
  - 원인 정확히 확정 (A/B/C 중)
  - 분할된 paragraph border 의 세로선이 col_bottom 을 넘지 않도록 수정
  - exam_kor.hwp 모든 페이지에서 PDF 와 일치하는 길이로 그려지는지 확인
  - 회귀 테스트: 표 외곽 테두리(원래 expand_clip 의 의도) 가 잘리지 않는지 확인
- **제외**:
  - body-clip 의 일반적인 정책 변경 (표 누락 방지 정책은 유지)
  - 다른 컨트롤(쉐이프, 그림 등) 의 bbox 처리

## 5. 검증 기준

1. `exam_kor.hwp` 의 page 2/5/8/15 세로선이 PDF 와 일치하는 위치에서 종료
2. body-clip 이 페이지 영역 밖으로 확장되지 않음 (page 8 의 1671 → 1423 이하로 축소)
3. 기존 표 외곽 테두리 테스트(expand_clip 도입 시점의 회귀 케이스) 통과
4. 다른 샘플 (`samples/` 내 주요 문서) 에서 새로운 회귀 없음
5. `cargo test` 통과
