# Task #452: 단락 마지막 줄 trailing line_spacing 누락 수정 — 수행계획서

## 배경 / 증상

`samples/exam_kor.hwp` 1페이지 좌측 단에서 pi=1("밑줄 긋기는 일상적으로...") → pi=2("통상적으로 독자는...") 전환 시 줄 간격이 단락 내 줄 간격보다 좁게 렌더링됨.

### 측정 (디버그 오버레이 SVG, 1페이지 좌측 단)

| 구간 | 측정 step | 기대 step | 차이 |
|------|-----------|-----------|------|
| pi=1 단락 내 (line0→line9) | 24.51 px | 24.51 px (lh+ls=1838 HU) | 0 |
| pi=1.line9 → pi=2.line0 | **15.34 px** | 24.51 px | -9.17 px (= 1 ls = 688 HU) |
| pi=2 단락 내 | 24.51 px | 24.51 px | 0 |
| pi=2.line4 → pi=3.line0 | 24.51 px | 24.51 px | 0 (우연 정상) |

PDF (한컴 렌더링) 은 12.96 pt 균일 stepping 으로 모든 줄이 같은 간격. HWP 파일 LineSeg vpos 도 1838 HU 균일. 문제는 rhwp 렌더링에만 존재.

## 원인

`src/renderer/layout/paragraph_layout.rs:2511-2520`:

```rust
let is_para_last_line = cell_ctx.is_none()
    && line_idx + 1 == end
    && end == composed.lines.len();
if (is_cell_last_line && cell_ctx.is_some()) || is_para_last_line {
    y += line_height;                          // ← trailing ls 누락
} else {
    let line_spacing_px = hwpunit_to_px(comp_line.line_spacing, self.dpi);
    y += line_height + line_spacing_px;
}
```

- Task #332 에서 "일반 문단의 마지막 visible 줄: trailing ls 제외" 로 도입된 분기.
- `layout_paragraph` 의 반환 `y_offset` 이 다음 문단의 `y_start` 가 되므로 (`layout.rs:574, 597, 606, 1811`), 단락 마지막에서 ls 가 빠지면 다음 단락 첫 줄이 ls 만큼 위로 당겨짐.
- pi=2→pi=3 가 정상으로 보이는 것은 `composed.lines.len()` 이 LineSeg 수와 달라 `is_para_last_line` 이 false 로 잡힌 우연. 매우 불안정한 동작.

## 목표 (옵션 A)

`is_para_last_line` 분기를 제거하고 모든 줄에서 `y += lh + ls` 적용 → 단락 경계 stepping 을 단락 내 stepping 과 동일하게 통일.

단, **셀 내 마지막 문단 (`is_cell_last_line && cell_ctx.is_some()`)** 은 표 셀 높이 계산과 직결되므로 **분기 유지**. 본 수정의 범위는 **본문 단락 (cell_ctx is None)** 의 마지막 줄 trailing ls 만.
