# Task #455 수행 계획서 — 인라인 글상자(tac=true + wrap=TopAndBottom) 본문 누락 수정

## 1. 배경

`samples/exam_kor.hwp` 페이지 2 좌측 단의 문단 pi=33 line 2 에서, 인라인 글상자(사각형, 내부 텍스트 "개화") 주변의 본문 39자(offset 35..73)가 모두 누락되어 글상자 안 "개화" 두 글자만 단 가운데에 표시된다.

원인은 글상자가 `treat_as_char=true` 와 `wrap=TopAndBottom` 을 동시에 가지고 있을 때, 렌더러가 `wrap=TopAndBottom` 분기를 우선해서 해당 줄의 본문 텍스트 흐름을 스킵하는 것으로 추정된다. HWP 표준에서는 `treat_as_char=true` 가 우선이며 이 경우 wrap 은 무시되어야 한다.

## 2. 목표

- pi=33 line 2 본문 39자(ts=35..73)가 정상적으로 렌더되도록 수정.
- 글상자 자체는 line 2 안에 인라인 글자처럼 배치(현 위치 x≈428.8 유지).
- 다른 문서/케이스에 회귀가 없도록 동일 패턴(tac=true + wrap≠None)을 가진 글상자 처리 경로를 일관되게 정규화.

## 3. 비목표

- 글상자 외관 스타일(테두리/채우기) 정밀 재현은 본 타스크에서 다루지 않는다.
- `treat_as_char=false` 인 글상자 wrap 처리는 변경하지 않는다.

## 4. 검증 방법

1. `cargo build --release` 후 `rhwp export-svg samples/exam_kor.hwp` 실행.
2. `output/svg/exam_kor/exam_kor_002.svg` 좌측 단 y≈321 줄에 본문 39자가 보이는지 확인.
3. python 스크립트로 좌측 단 y 좌표별 글자 수를 계산해 line 2 글자 수가 35→ ~39 로 늘어났는지 확인.
4. `cargo test` 전체 통과.
5. 페이지 수 회귀(20페이지 유지) 확인.
6. Task #332/#409/#412/#452 등 직전 회귀 케이스 영향 점검.

## 5. 주요 변경 후보

- `src/renderer/composer.rs` — Shape tac 컨트롤 등록 시 wrap 무시.
- `src/renderer/layout/shape_layout.rs` — 글상자 분기에서 `treat_as_char` 시 wrap=None 으로 정규화.
- `src/renderer/layout.rs` — `prev_has_overlay_shape` 등 wrap 기반 분기에서 tac 인 경우 제외 확인.
- `src/renderer/height_measurer.rs` — line height 계산이 wrap 분기를 거치는 경우 점검.

## 7. 위험 요소

- `tac=true + wrap=TopAndBottom` 패턴이 다른 문서에서 다른 의미로 쓰일 수 있다 → 회귀 발생 시 수정 범위 축소.
- 글상자 line height(lh=1417) 가 본문 line height(lh=1150) 보다 큰데, 이를 line 2 안에서 어떻게 정렬할지 결정 필요(top/center/baseline).
