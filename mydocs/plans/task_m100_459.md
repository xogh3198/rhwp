# Task #459 수행계획서: 다단 후속 페이지에서 LINE_SEG vpos-reset 단 경계 미인식

## 1. 배경

`samples/exam_kor.hwp` 페이지 2의 좌측 단 하단에서 `pi=39` 문단이 col_bottom을 39px 초과하여 그려짐. HWP 원본은 LINE_SEG의 `vertical_pos` 리셋(ls[2]:vpos=0)으로 단 경계를 명시했으나 렌더러가 이를 무시.

### 증거
- `pi=39` LINE_SEG: ls[0..2] vpos=86417,88255 (좌측 단), ls[2..7] vpos=0,1838,3676,5514,7352 (우측 단)
- 현재 렌더링: 좌측 단 lines=0..4 (4줄), 우측 단 lines=4..7 (3줄) → 좌측 단 39px overflow
- 정답: 좌측 단 lines=0..2 (2줄), 우측 단 lines=2..7 (5줄)

## 2. 원인

`src/renderer/pagination/engine.rs:607`의 가드:
```rust
if st.col_count > 1 && st.current_column == 0 && st.on_first_multicolumn_page {
    Self::detect_column_breaks_in_paragraph(para)
}
```
`on_first_multicolumn_page`는 `state.rs:218`에서 새 페이지 시작 시 false로 리셋. 다단 구역이 여러 페이지에 걸칠 경우 후속 페이지에서 단 경계 감지가 작동하지 않음.

`git log -G "on_first_multicolumn_page"`는 최초 커밋 한 번만 나옴 → 가드의 도입 이유는 코드 주석이나 회귀 케이스 기록이 없는 초기 설계상 보수적 가드로 추정.

## 3. 수정 방향 (옵션 1)

`engine.rs:607`에서 `&& st.on_first_multicolumn_page` 제거.

```rust
let col_breaks = if st.col_count > 1 && st.current_column == 0 {
    Self::detect_column_breaks_in_paragraph(para)
} else {
    vec![0]
};
```

근거:
- `detect_column_breaks_in_paragraph`는 vpos 감소만 감지하므로 단일 컬럼 문단에는 영향 없음 (breaks=[0]만 반환).
- `current_column == 0`은 유지 — 이미 우측 단에 있는 paragraph는 단 경계가 의미 없음.
