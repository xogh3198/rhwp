## 3. 파서 처리 메커니즘

### 3.1 AutoNumber U+FFFC → "  " 변환

HWP3 캡션 텍스트에서 AutoNumber 위치는 U+FFFC(OBJECT REPLACEMENT CHARACTER)로 저장된다.
HWP5/HWPX 는 공백 두 개(`"  "`)를 placeholder로 사용한다.

**파서 처리**: `src/parser/hwp3/mod.rs` 에서 텍스트 구성 시 `'\u{fffc}'` → `"  "` 치환.
→ 렌더러의 기존 `"  "` 패턴 검색이 정상 작동. 렌더러 변경 불필요.

### 3.2 혼합 단락 LINE_SEG 높이 보정 (그림 겹침 해소)

**문제**: HWP3 혼합 단락(텍스트 + Para-relative TopAndBottom 그림)에서
`document.rs`의 `need_vpos_recalc`가 LINE_SEG vpos를 순차 누적으로 덮어쓰므로
줄 6-12의 vpos가 그림 구역[112px, 596px]에 위치 → 텍스트와 그림이 겹침.

**해소 원리**: 파서에서 마지막 "그림 위쪽" LINE_SEG의 `line_height`를 그림 하단까지의
거리로 확장한다. `compose_paragraph`가 이 값을 `ComposedLine.line_height`로 복사하므로
렌더러의 순차 `y += line_height`가 자동으로 그림 하단으로 점프한다.

**수치 검증 (pi=76 기준)**:
```
원래:  seg[5] pos=8000 HU, line_height=1000, line_spacing=600
수정:  seg[5] line_height=36700 HU (= fig_bottom 44700 - pos 8000)
       seg[5] text_height=0   (document.rs advance=lh+ls=36700 확보)
       seg[5] line_spacing=0

렌더러 sequential y += line_height:
  줄 0-4: y_start → y_start+106.7px ✓
  줄 5:   y_start+106.7px, line_height=489.3px → y+=489.3
  줄 6:   y_start+596px (그림 하단) ✓
  줄 7-12: y_start+617.3 ~ 736.9px ✓

document.rs 파생 검증:
  seg[5] advance = th+ls = 0+0→ lh+ls = 36700+0 = 36700 HU
  전체 advance 합 = 8000+36700+11200 = 55900 HU → 올바른 단락 높이 ✓
  seg_lh_total = 55900 > obj_total 44700 → 추가 보정 없음 ✓
```
