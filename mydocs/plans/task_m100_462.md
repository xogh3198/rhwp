# Task #462 수행계획서: TAC Picture 인라인 문단 line advance 누락

## 1. 배경

`samples/exam_kor.hwp` 페이지 1 우측 단 pi=25 (빈 문단 + TAC `Control::Picture`) 직후 pi=26 사이 여백이 약 20.5px 부족. PDF 와 비교 시 그림 바로 아래에 본문이 너무 가깝게 붙음.

### 좌표 증거
- pi=25 이미지 bottom: y=1113.58
- pi=26 line_top: y=1113.57 (실제 갭 ~0)
- HWP LINE_SEG: `lh=24525, ls=688` HU → 합 `25213 HU = 336.17 px`
- 이미지 자체 높이: `23675 HU = 315.67 px`
- 누락분: `leading 11.33 + line_spacing 9.17 = 20.51 px`

## 2. 원인

`src/renderer/layout.rs:2724` (TAC `Control::Picture` 인라인 경로):
```rust
result_y = pic_y + pic_h;
```
이미지 박스 높이만 사용하고 LINE_SEG 의 `lh + ls` 를 무시. layout.rs:2727 의 `already_registered` 분기도 동일 로직.

TAC Shape 보정 (1929-1958) 은 `Control::Shape` 만 매칭 → Picture 에는 적용 안 됨. 추가로 보정이 작동해도 `shape_bottom = para_start + effective_h` 만 사용하고 `line_spacing` 누락 (`spacing_after` 만 추가).

## 3. 수정 방향 (옵션 A 채택)

`layout.rs:2724` + 2727 의 두 지점에서 LINE_SEG 기반 advance 사용:
```rust
let line_advance = para.line_segs.first()
    .map(|ls| hwpunit_to_px(ls.line_height + ls.line_spacing, self.dpi))
    .unwrap_or(pic_h);
result_y = pic_y + line_advance.max(pic_h);
```

`max(pic_h)` 는 LINE_SEG 가 잘못 작은 값을 가질 때의 안전장치 (이미지가 잘리지 않도록).
