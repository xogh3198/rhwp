# Task #463 수행계획서

## 제목

exam_kor 14p: 본문 외곽선이 셀 단락 leakage 로 4개 박스로 쪼개지는 현상 수정

## 배경

`samples/exam_kor.hwp` 14페이지에서 (가) 학생 대화 전체가 동일한 `border_fill_id=7` 을 공유하지만, rhwp SVG 출력은 4개의 분리된 외곽선 박스로 그려진다. PDF 정답은 단일 외곽선 박스 1개.

분석 결과:

- `border_fill[7]` 의 4면 모두 Solid w=1 (정상 외곽선 정의)
- ParaShape[40] attr1 의 외곽선 게이팅 비트는 없음 (가설 A/B/C 모두 기각)
- **진짜 원인**: 인용 따옴표 ｢｣ 표시용 3×2 표의 **셀 안 단락들이 본문 단락과 같은 `para_border_ranges` 큐에 push 되어 merge 를 깨뜨림**
  - 셀 단락 bf=4 (시그니처 None) → 본문 bf=7 그룹 단절
  - 셀 단락 bf=6 (시그니처 Solid w=1, bf=7 과 동일) → 본문 그룹이 셀 좌표(x≈533, w=0)로 흡수되어 다음 본문이 다시 새 그룹 시작

## 목표

좌측 단의 22개 본문 대화 문단이 단일 외곽선 그룹으로 병합되어 PDF 와 동일하게 1개 큰 박스로 렌더링되도록 수정한다.

## 작업 범위

### In-scope

- `src/renderer/layout/paragraph_layout.rs`: `para_border_ranges` push 시 cell context 게이팅 추가
- `src/renderer/layout.rs`: merge 로직은 손대지 않음 (입력 큐가 정상화되면 자동 해결)
- exam_kor 14p 좌측 단 외곽선이 1개 큰 박스로 합쳐지는지 SVG 비교 검증

### Out-of-scope

- 표 셀 외곽선 렌더링 (별도 경로 `table_layout` 으로 처리됨, 회귀 검증만)
- ParaShape attr1 비트 28~31 해석 추가 (이번 이슈와 무관)
- 다른 페이지의 박스 회귀는 회귀 테스트 통과 여부로 확인

## 검증 기준

1. exam_kor 14p 좌측 단 SVG 의 본문 외곽선 사각형이 **1~2개**(상단 [38~42] 헤더 박스 + (가) 대화 박스)로 줄어든다.
2. 본문 박스의 x/y/w/h 가 PDF 와 시각적으로 일치한다 (≤5px 오차).
3. 표 셀 외곽선(인용 따옴표 ｢｣) 은 영향 없음.
4. 회귀 테스트 (`cargo test`, integration_tests) 통과.
5. clippy `-D warnings` 통과.

## 리스크

- `cell_ctx` 가 있는 단락이 본문 외곽선에서 제외되더라도 **표 안 셀 단락이 별도 외곽선을 그려야 하는 정상 케이스**가 있으면 그쪽이 깨질 수 있다 → integration_tests 의 표 외곽선 케이스로 회귀 확인.
- merge 로직이 stroke 시그니처로만 병합하는 정책은 다른 케이스에서도 잠재적 문제. 본 이슈에서는 입력 정상화로 해결하지만 추후 별도 이슈 검토 권장.
