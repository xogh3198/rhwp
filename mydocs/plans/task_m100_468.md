# Task #468 수행계획서

## 제목

exam_kor 6p 좌측 단 [18~21] passage 박스 하단 줄 — cross-column 박스 연속 미감지

## 배경

`samples/exam_kor.hwp` 6페이지 좌측 단 (col 0) 의 [18~21] passage 박스가 col_bottom 근처에서 하단 stroke 가 그려져 박스가 닫혀 보임. PDF 는 박스가 col 1 / 페이지 7 까지 이어지므로 col 0 하단 stroke 미렌더.

### 현상

```xml
<rect x="128.50" y="1182.48" width="400.61" height="235.10"
      fill="none" stroke="#000000" stroke-width="0.5"/>
```

→ 4면 모두 stroke. col 0 마지막 paragraph (pi=194, FullParagraph) 의 `is_partial_end = end < composed.lines.len()` 이 false → bottom 그려짐.

### 박스 구성

| pi | h | 위치 | text | bf_id |
|----|---|------|------|-------|
| 192 | 72.0 | col 0 page 6 | [앞부분의 줄거리]... | X |
| 193 | 61.3 | col 0 page 6 | 원수가 대경하여... | X |
| 194 | 15.3 | col 0 page 6 | "어서 바삐..." | X (col 0 마지막) |
| 195 | 46.0 | col 1 page 6 | 한데 왕비 크게 노하여... | X (계속) |

## 원인

`paragraph_layout.rs:2527-2528`:

```rust
let is_partial_start = start_line > 0;
let is_partial_end = end < composed.lines.len();
```

이 플래그는 **단일 paragraph 내부 split** (PartialParagraph) 만 검출. 별개 paragraph 가 sequential 하게 같은 bf_id 를 가지며 컬럼 경계로 분리된 케이스 미감지.

## 수정 방향 (예상 — 옵션 A: sequential next/prev paragraph 검사)

순차 paragraph 의 bf_id 비교는 컬럼 lookahead 없이 가능:

- **partial_end 검출**: merge 그룹의 마지막 paragraph 의 para_index = P 일 때, paragraph[P+1] 이 같은 bf_id 면 → 박스 연속 → partial_end = true
- **partial_start 검출**: merge 그룹의 첫 paragraph 의 para_index = P 일 때, paragraph[P-1] 이 같은 bf_id 면 → partial_start = true

구현 위치 후보:
1. **`paragraph_layout.rs:2527+` (push 시점)** — push 직전에 next/prev paragraph 의 bf_id 확인
2. **`layout.rs::build_single_column` (merge 후)** — merge 그룹별로 첫/마지막 paragraph 의 sequential 인접 bf_id 확인

옵션 2 가 깔끔 (merge 결과 기반, 정합성 보장).

### 추가 고려

- **별개 박스인데 같은 bf_id 인 경우**: paragraph 들 사이에 같은 bf_id 가 없는 paragraph 가 끼면 별개 박스. merge 로직이 이미 구분하므로 그룹 단위로 판단 → 이슈 없음.
- **시작/끝 박스의 partial 마킹**: 두 박스가 같은 bf_id 라도 sequential 이 아니면 partial 마킹 안 됨 (정상).
