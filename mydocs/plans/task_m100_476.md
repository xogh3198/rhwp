# Task M100 #476 수행 계획서

## 1. 이슈 요약

GitHub Issue: [#476](https://github.com/edwardkim/rhwp/issues/476)
브랜치: `local/task476`
마일스톤: M100 (v1.0.0)

**증상**: `samples/21_언어_기출_편집가능본.hwp` 페이지 12에서 21번 본문 paragraph(`pi=238`) 안의 인라인 박스 "배너지와 뒤플로"가 22번 문제 위쪽 빈 영역(y≈742px)에 잘못 그려진다. 박스 안 텍스트도 같이 쏠려 "왼쪽 박스 아래 '배너지와 뒤플로'"가 출력된 것처럼 보인다.

**부수 영향**: 페이지 12 단 0 사용량 1012px vs 한컴 기준 1219px(diff -206.8px) — 23번 답안 일부가 단 밖으로 밀려난다.

## 2. 근본 원인 (확정)

페이지간 `inline_shape_positions` 단절로 인한 fallback 위치 오류:

| 단계 | 동작 |
|------|------|
| (1) | `pi=238`은 길이 871자·26줄, 페이지 11~12에 걸쳐 분할(PartialParagraph) |
| (2) | 인라인 박스(treat_as_char=true)가 char=0 (line 0)에 위치 |
| (3) | 페이지 11 `paragraph_layout`이 line 0 처리 → `set_inline_shape_position(sec, 238, 0, x, y)` 등록 |
| (4) | 그러나 `PageRenderTree.inline_shape_positions`는 **페이지마다 새로 생성**됨 (`render_tree.rs:730`) |
| (5) | paginator가 `PageItem::Shape{pi=238}`을 **페이지 12에도** 등록 |
| (6) | 페이지 12 `layout_column_shapes_pass` → `get_inline_shape_position()` → `None` → `compute_object_position` fallback (`shape_layout.rs:218`) |
| (7) | Shape 위치 "가로/세로=문단 오프셋=0,0"이라서 paragraph 238의 페이지 12 시작 y(742.45)에 박스를 잘못 다시 그림 |

디버그 로그로 검증 완료(자세한 출력은 이슈 #476 본문 참조).

## 3. 수정 방향 비교

| 옵션 | 변경 범위 | 효과 | 리스크 |
|------|----------|------|--------|
| **A** | paginator 인라인 Shape 페이지 라우팅 | 박스가 속한 line의 페이지에만 PageItem::Shape 등록 | 페이지 분할 로직 변경. 회귀 위험 큼 |
| **B** | inline_shape_positions를 Document 단위로 공유 | 페이지간 좌표 공유 | 페이지마다 좌표계 다름 — 부적절 |
| **C** | treat_as_char Shape는 paragraph_layout에서 직접 렌더, PageItem::Shape 미등록 | 가장 깔끔. inline_pos 우회 패턴 폐기 | z-order 정렬 패스 호환 검토 필요 |
| **D** | inline_pos=None + 이전 페이지에서 시작된 PartialParagraph인 경우 박스 렌더 스킵 | 작은 표면적, 회귀 차단 즉시 | 근본 원인 미해결, 임시 패치 |

### 추천 전략

**D (임시 차단) → A (본질 수정)** 단계적 접근.

D 단계에서 즉시 회귀 차단하고 정상성을 재확립한 뒤, A로 paginator 라우팅을 정정하여 깔끔한 구조로 끝낸다. C는 z-order 처리와 layout_column_shapes_pass의 다른 의존성 검토가 필요해 본 타스크 범위를 넘는다(별도 리팩터 타스크 후보).
