# Task #477 수행 계획서 — 표 셀 안 그림이 셀 폭 초과 시 클램프 미적용

## 배경

[이슈 #477](https://github.com/edwardkim/rhwp/issues/477) 정황:
- `samples/k-water-rfp.hwp` 16쪽 두 번째 표 (pi=186) 셀 안 그림이 셀 경계 초과
- 그림 폭 (165.0 mm) > 셀 폭 (164.0 mm) — 1 mm 초과
- 한컴 에디터는 셀 경계로 자동 클램프, rhwp 는 원본 그림 폭 그대로 렌더링

## 본질 — `layout_picture` 호출부의 컨테이너 정합 결함

`layout_picture` 함수 자체에는 컨테이너 초과 시 비율 유지 축소 로직이 있음 (`picture_footnote.rs:39-48`):

```rust
if container.width > 0.0 && pic_width > container.width {
    let scale = container.width / pic_width;
    pic_width = container.width;
    pic_height *= scale;
}
```

하지만 호출부에서 **`pic_area.width = pic_w`** (그림 폭 그대로) 로 컨테이너를 만들어 전달:

```rust
// table_layout.rs:1533-1538 (TAC 표 셀 안 그림)
let pic_area = LayoutRect {
    x: inline_x, y: tac_img_y,
    width: pic_w,   // ← 그림 폭 자체
    height: pic_h,  // ← 그림 높이 자체
};
self.layout_picture(tree, &mut cell_node, pic, &pic_area, ...);
```

→ `layout_picture` 내부의 `pic_width > container.width` 비교는 항상 false (자기 자신과 비교) → 클램프 무효화.

## 영향 영역

같은 결함 패턴이 있는 호출부 4개소:

| 파일 | 라인 | 정황 |
|------|------|------|
| `layout/table_layout.rs` | 1539 | TAC 표 셀 안 단독 이미지 |
| `layout/table_partial.rs` | 715 | TAC 분할 표 셀 안 단독 이미지 |
| `layout/shape_layout.rs` | 1523 | TAC 도형 컨테이너 안 그림 |
| `layout/shape_layout.rs` | 1533 | TAC 도형 컨테이너 안 그림 (정렬 분기) |

비-TAC 호출부 (`table_layout.rs:1547-1563`, `table_partial.rs:720-725`, `layout.rs:572,824`) 는 이미 셀 영역 (`inner_area`/`cell_area`) 를 전달하여 클램프 정상 작동.

## 정정 방향

호출부에서 **셀 안쪽 영역 (inner_area / pic_container)** 을 컨테이너로 전달.

```rust
// 정정 후 (예시)
let pic_area = LayoutRect {
    x: inline_x, y: tac_img_y,
    width: inner_area.width,    // ← 셀 안쪽 폭
    height: inner_area.height,  // ← 셀 안쪽 높이
};
self.layout_picture(tree, &mut cell_node, pic, &pic_area, ...);
```

또는 호출부에서 미리 클램프:

```rust
let clamped_w = pic_w.min(inner_area.width);
let clamped_h = pic_h * (clamped_w / pic_w).min(1.0);
```
