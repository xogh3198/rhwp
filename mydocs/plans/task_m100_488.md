# Task M100 #488 — 수식 토크나이저 폰트 스타일 키워드 prefix 분리

## 1. 배경

`samples/exam_science.hwp` 페이지 1의 화학 수식이 `rmK`, `rmCa`, `rmmol`, `itl`, `itaq` 등 raw 식별자로 출력되고 있음. 한컴 hwpeq 문법에서 `rm` / `it` / `bold` 폰트 스타일 키워드는 다음 식별자에 공백 없이 붙어 쓸 수 있는데, 현재 토크나이저가 이를 분리하지 못함.

`samples/pdf/hwp2022/exam_science.pdf`(한컴 정답 PDF)와 비교 시, 페이지 1 표 #3(이온 결합 화합물)뿐 아니라 페이지 1 본문 전반에 걸쳐 광범위하게 발생.

## 2. 목표

- `rm` / `it` / `bold` 폰트 스타일 키워드가 식별자 prefix로 사용된 경우 토크나이저가 키워드를 분리하여 정상 토큰으로 처리.
- `samples/exam_science.hwp` 페이지 1의 K⁺, X⁻, Ca²⁺, O²⁻, mol, H₂O(l), KOH(aq) 등이 정답 PDF와 동일하게 렌더링.
- 기존에 정상이던 다른 샘플의 수식·일반 식별자에 회귀 없음.

## 3. 변경 범위

| 파일 | 변경 내용 |
|------|----------|
| `src/renderer/equation/tokenizer.rs` | `read_command()` 에서 폰트 스타일 키워드(`bold`/`it`/`rm`) prefix 매치 시 거기서 토큰 종료 |
| `src/renderer/equation/tokenizer.rs` (tests) | prefix 분리 단위 테스트 추가 (`rmK`, `rmCa`, `rmmol`, `itl`, `itaq`, `boldX`, 그리고 일반 명령 회귀 테스트) |

수식 파서·렌더러·심볼 매핑은 변경 없음. 토크나이저 단계에서 분리만 정확히 하면 기존 파서가 정상 동작.

## 4. 검증 계획

### 4.1 단위 테스트 (cargo test)

`src/renderer/equation/tokenizer.rs` 의 기존 테스트 패턴(`tests` 모듈)에 다음 케이스 추가:

- `rmK ^{+}` → `["rm", "K", "^", "{", "+", "}"]`
- `rmCa ^{2+}` → `["rm", "Ca", "^", "{", "2", "+", "}"]`
- `1\`rmmol` → `["1", "WS", "rm", "mol"]`
- `LEFT ( itaq RIGHT )` → `["LEFT", "(", "it", "aq", "RIGHT", ")"]`
- 회귀: `boldsymbol` 같은 별개 키워드는 `bold` 분리되지 않도록(없으면 그대로 유지). 단, `bold` 자체는 FONT_STYLES이므로 `boldX` → `["bold", "X"]` 로 분리.
- 회귀: `MATRIX`, `LEFT`, `over`, `sim` 등 기존 명령은 현재처럼 한 토큰으로 유지.

### 4.2 SVG 시각 검증

- `cargo build --release && ./target/release/rhwp export-svg samples/exam_science.hwp -p 0 -o output/svg/exam_science_488/`
- 결과 SVG 검사: `rmK`/`rmCa`/`rmmol`/`itl`/`itaq` 등의 raw prefix 문자열이 텍스트로 남지 않을 것 (`grep -E ">rm[A-Z]<\|>it[a-z]<" output/svg/exam_science_488/*.svg` → 0건)
- `samples/pdf/hwp2022/exam_science.pdf` 페이지 1과 비교하여 K⁺, X⁻, Ca²⁺, O²⁻, mol, H₂O, KOH(aq) 등이 정답과 동일하게 표시.

### 4.3 회귀 검증 (광범위 샘플)

`samples/` 내 주요 HWP/HWPX 파일에 대해 수정 전·후 SVG 출력을 비교:
- `samples/exam_kor.hwp` (있을 경우)
- `samples/k-water-rfp.hwp`
- `samples/synam-001.hwp`
- 기타 `samples/` 직속 `.hwp` / `.hwpx`

각 파일의 페이지 1 SVG에서 수식 토큰이 깨지지 않았는지 spot check (특히 `rm`/`it` 관련 이슈 외 영역).

피드백 메모리(`feedback_essential_fix_regression_risk.md`)에 따라, 본질 정정에 가까운 변경은 광범위 샘플 + 한컴 정답지 검증 필수. 이 작업은 토크나이저 단의 키워드 분리만 다루므로 레이아웃·페이지네이션과는 직교 영역이지만, 수식 출력 변화는 시각적으로 명확히 드러나므로 페이지 1 단위 비교를 다수 샘플에 대해 수행한다.

   - 단계별 보고서: `mydocs/working/task_m100_488_stage2.md`

## 6. 위험 요소

- **(낮음) 기존에 우연히 통과하던 케이스 회귀**: 토크나이저가 식별자 안에 `rm`/`it`/`bold`가 들어간 단어를 분리해버릴 가능성. hwpeq 문법상 이런 식별자는 거의 등장하지 않으나(`arm`, `firm` 등은 변수명으로 쓰이지 않음), 회귀 검증으로 확인.
- **(낮음) 키워드 길이 우선순위**: `bold` 5자, `it` 2자, `rm` 2자. `it`/`rm` 동시 매치는 불가능 (서로 다른 글자). `bold` 5자 vs 짧은 것은 충돌 없음.
