## 2. 증상

`samples/exam_science.hwp` 페이지 1 컬럼 1 (단 1, 우측 컬럼) 의 5번 문제 본문(pi=21, "5. 그림은 밀폐된 진공 용기에…") 이 다음 두 결함을 보인다.

1. **그림에 의한 본문 가림** — 첫 줄의 "용기에 H₂O(l)을" 부분이 그림 영역에 들어가서 보이지 않음.
2. **비정상적으로 큰 단어 간격** — 첫 4~6줄이 풀컬럼 너비로 justify 되어 어절 사이 간격이 60~100px.

정답 PDF (`samples/pdf/hwp2022/exam_science.pdf`)에서는 그림이 좌측, 본문이 우측에 어울림 배치되어 정상 표시.

`Control::Picture` / `Control::Shape` 는 동일한 wrap=Square 일지라도 위 조건문이 매칭되지 않아 wrap 영역 처리 없이 풀컬럼에서 정상 흐름으로 렌더 → LINE_SEG 의 `cs/sw` 가 페이지네이션·높이 계산에는 반영되지만 실제 텍스트 그리기 위치에는 미반영.

## 4. 수정 방향

### 4.1 핵심 원칙

- **HWP LINE_SEG 의 `cs/sw` 는 한컴 2010/2020 이 인코딩한 정답** — 6줄 모두 sw=19592 로 그림 너비만큼 좁아짐. 새 측정 없이 **이 값을 그대로 사용**한다.
- **표 Square wrap 처리(Task #362/#439/#463)와 동일 패턴** 으로 Picture/Shape 까지 확장한다.
- 휴리스틱·허용오차 도입 금지. `wrap=Square && !treat_as_char` 일 때 항상 LINE_SEG 의 `cs/sw` 를 따른다.

### 4.2 수정 영역

| 파일 | 함수 / 위치 | 변경 |
|------|-----------|------|
| `src/renderer/layout.rs` | `:2399` (`if let Some(Control::Table(t))` ...) | `Control::Picture(p)`, `Control::Shape(s)` 분기 추가 |
| `src/renderer/layout.rs` | `:2634` (분할 표 어울림) | Picture 는 분할 없으므로 단순 케이스만 추가 |
| `src/renderer/layout.rs` | `compute_square_wrap_tbl_x_right` | Picture 용 등가 함수 신설 또는 일반화 |
| `src/renderer/typeset.rs` | `:608` (wrap zone 활성화) | Picture/Shape Square 도 wrap zone 시작 트리거 |
| `src/renderer/typeset.rs` | `:1466` (Square wrap 표 식별) | Picture/Shape 까지 확장 |

### 4.3 회귀 위험

- **표 Square wrap (Task #362/#439/#463) 회귀 가능성**: 같은 코드 패스를 공유하므로 분기 조건이 잘못 추가되면 표 케이스가 깨질 수 있음. → Picture/Shape 분기는 Table 분기와 **and-disjoint** 하게 추가.
- **TopAndBottom Picture (Task #409 v2) 회귀 가능성**: 별도 패스이므로 직접 영향 없으나, `compute_body_wide_top_reserve_for_para` 등 Picture 일반 처리에 영향 없는지 확인.
- **TAC Picture 회귀 가능성**: `treat_as_char=true` 인 그림은 글자처럼 동작 → 본 변경 대상 아님 (`!common.treat_as_char` 조건 유지).
- **샘플 회귀 검증**: `samples/` 의 모든 HWP 파일 SVG 출력 byte 비교, 차이 발생 페이지 시각 점검.
