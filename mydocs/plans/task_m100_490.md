이 분기는 Task #287 (큰 디스플레이 수식이 자체 LINE_SEG 가질 때 col_area.y 고정 결함) 처리용으로 도입됐으나, **paragraph alignment 적용을 누락**. 동일 함수의 텍스트 runs 케이스 (`paragraph_layout.rs:1200~1211`) 는 `Alignment::Center | Distribute | Right` 분기로 x_start 보정. 빈 runs 케이스는 보정 없이 좌측 고정.

ParaShape ps_id=3 (셀 paragraph) 의 alignment 가 Center 일 때, 수식만 있는 셀은 좌측에 나타남.

## 4. 수정 방향

### 4.1 핵심 원칙

- 텍스트 runs 케이스의 alignment 보정 패턴을 빈 runs 케이스에도 동일 적용
- `available_width`, `alignment` 변수는 이미 함수 스코프 안에 존재 → 추가 의존성 없음
- 표 처리, 일반 paragraph, 인라인 수식 모두에 동일 시멘틱

### 4.2 수정 영역

| 파일 | 위치 | 변경 |
|------|------|------|
| `src/renderer/layout/paragraph_layout.rs` | `:2232` 직전 | `line_tac_width` 계산 + `align_offset` 산출 + `inline_x` 에 더하기 |

### 4.3 회귀 위험

- **Justify/Left ParaShape**: `align_offset = 0.0` (match 의 `_ => 0.0`) → 변경 없음 (기존 동작 유지)
- **Center 셀의 텍스트+수식 혼재 줄**: comp_line.runs 가 비어있지 않으므로 본 분기 미진입 → 영향 없음
- **수식 표시 위치 (Task #287 fix 영역)**: alignment 적용은 x 만 변경, y/height 미변경 → 회귀 없음
