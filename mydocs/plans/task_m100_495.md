## 1. 배경

`samples/exam_science.hwp` 2페이지 좌측 단의 7번 문제 박스(1×1 표) 내부의 한 줄이 동일 글자로 두 번 렌더링되어 시각적으로 겹침. PDF(`samples/pdf/hwp2022/exam_science.pdf` 페이지 2) 와 비교 결과 본 라인은 한 번만 그려져야 한다.

박스 자체 위치/크기는 정상이며, 박스 외부 본문(pi=34, 표 pi=35, 8번 pi=37) 좌표도 dump-pages 기준 정상이다. **문제는 박스 셀 내부 p[1] 문단의 텍스트 발행 경로에 한정**된다.

### 2.2 IR 구조

`rhwp dump -s 0 -p 33` 결과 박스 셀 p[1]:

```
cc=2 ctrls=2 text_len=84
ls[0] vpos=1610 lh=1150
ls[1] vpos=3220 lh=1716
ctrl[1] 사각형: tac=true, wrap=TopAndBottom
```

`tac=true`(글자 취급) ∧ `wrap=TopAndBottom`(블록 취급)인 모순된 사각형 컨트롤이 트리거.

## 3. 가설

composer (또는 render_tree 빌더)가 `tac=true`인 사각형을 **인라인으로 한 번** 발행하고, 추가로 `wrap=TopAndBottom`을 만족시키기 위한 라인 재배치 경로에서 **호스트 라인을 한 번 더** 발행하여 동일 텍스트가 두 번 그려진다.

근거:
- y=219.01과 y=224.26의 글자 간격이 다름(~17px / ~20px) → 두 번의 layout이 서로 다른 segment_width 또는 char_overlap 으로 발행됨.
- p[1]에는 line_seg가 2개(ls[0]/ls[1]) 뿐이며, ls[0]의 vpos=1610 은 y=219.01 과 일치, ls[1]의 vpos=3220 은 y=240.48 과 일치 — 즉 IR이 두 번 그릴 의도는 아님.
- y=224.26은 IR의 어떤 lineseg vpos에도 대응하지 않음 → 별도 경로에서 추가 발행됨.
