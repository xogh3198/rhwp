### 1.2 결함 본질 (확정)

`src/renderer/equation/parser.rs:202-204` —
```rust
if cu == "OVER" {
    return EqNode::Empty;
}
```

`parse_command` 가 `OVER`/`ATOP` 을 단독 호출 시 `EqNode::Empty` 로 폐기한다. OVER/ATOP 의 중위 연산자 처리는 `parse_expression`(line 104-125) 와 `parse_group`(line 456-484) 에만 존재하며, **`parse_cases` / `parse_pile` / `parse_eqalign` 은 `parse_element` 를 직접 호출**하므로 OVER 가 분실된다.

**측정 (cargo test --lib probe_pi151_vs_pi165)**:
| | layout (px) | HWP (px) | scale_y |
|---|---|---|---|
| pi=151 (분수 없음) | 40.34 | 46.87 | 1.16 |
| pi=165 (분수 있음, 분실) | **40.49** | 66.27 | **1.64** |

pi=165 의 layout height 가 pi=151 과 동일 (40.49 vs 40.34) — 분수 손실 증거. 정상이면 분수 추가분 ~10~15 px 증가해야 함.

### 1.3 부수 결함 (영향 작음)

EqAlign 안의 leading `\n` (newline 후속) 이 `Text("\n")` 으로 파싱되어 width 7.7 px 차지. 본 이슈에서 우선 정정하나, 본질 결함 아님.
