**문서 정체 (작업지시자 확인)**: 고려대학교 복학신청서 양식. 누락된 그림은 **고려대학교 학교 로고** 이미지로, 양식 좌상단에 배치되어 양식의 발급 기관을 시각적으로 식별하는 기능을 수행한다. 한컴 출력 (`samples/복학원서.pdf`) 에서는 정상으로 나타난다.

본 결함의 시각 영향: 양식의 기관 식별 정보 누락 → 양식의 신뢰성/정합성 손상.

## 작업 목표

`samples/복학원서.hwp` 의 s0:pi=0 BehindText 고려대학교 로고 그림이 한컴 정답지와 동일한 위치 (단 오프셋 가로 2.3mm + 문단 오프셋 세로 3.0mm, 20.4×23.3mm) 에 정상 출력되도록 정정한다.

부수: 본 결함이 BehindText 그림 일반에 발생하는지 (다른 샘플) 또는 본 fixture 한정인지 (구역나누기 + 단정의 와 같은 컨트롤에 ci=2 로 attach 된 특수 케이스) 진단 후 적절한 가드 / 정정 적용.

**시각 판정 기준점**: 한컴 출력에서 양식 좌상단의 고려대학교 로고가 보이는 위치 + 크기 — rhwp SVG 출력에서도 동일하게 보여야 한다.

## 1차 진단 가설

### 가설 A: 빈 문단 (text_len=0) + BehindText 의 paint 가드 누락

`paragraph_layout.rs` 또는 `layout.rs::layout_shape_item` 에서 빈 문단의 paint 분기에서 BehindText 그림 emit 이 누락되었을 가능성. 메모리 `tac_picture_double_emit` 트러블슈팅의 정정 (Task #376/418) 이 빈 문단 + TAC=true 의 이중 emit 을 막는 가드를 추가했는데, 본 fixture 의 그림은 **TAC=false** 라서 가드와 무관해야 하는데, 가드 로직이 BehindText (TAC 무관 절대 위치) 도 차단하고 있을 가능성.

### 가설 B: 동일 문단의 다른 컨트롤 (구역나누기, 단정의) 이 그림 emit 을 차단

pi=0 은 controls=3 (구역나누기 + 단정의 + 그림) — `[0]` (구역나누기) 가 처리될 때 후속 controls 처리 중단 또는 paint 단계에서 그림이 emit 되기 전에 페이지 전환 로직이 그림을 흘려보낼 가능성.

### 가설 C: BehindText 그림의 paper 기준 절대 위치 처리 경로가 표 한정으로만 구현됨

`src/renderer/layout.rs:3295` 의 주석 "InFrontOfText/BehindText 표: paper 기준 절대 위치에 렌더링" 은 **표 한정**. 그림 (Picture) 의 BehindText 도 paper 기준 절대 위치로 그려져야 하는데 별도 경로가 있는지 또는 텍스트와 같은 vpos 누적 경로에 따라가다 paint 직전에 누락되는지 확인 필요.

### 가설 D: 단 오프셋 (`horz=Column, off=660`) + 문단 오프셋 (`vert=Para, off=840`) 좌표 산출 결함으로 페이지 밖 좌표로 계산되어 paint 가 skip

좌표가 음수 또는 페이지 밖으로 계산되면 SVG 출력에서 보이지 않음.
