## 본질 (1차 진단)

`samples/복학원서.hwp` 의 가운데 고려대학교 엠블렘 이미지에 한컴 워터마크 효과 (`effect=GrayScale + brightness=-50 + contrast=70`) 가 적용되어 있다. **CLI SVG 는 정상 적용** 되나 (`<filter rhwp-img-grayscale>` + `<filter rhwp-img-bc-b-50c70>` slope=1.7 intercept=-0.85), **Web Canvas 는 원본 컬러로 출력** → 흐릿한 워터마크 효과 누락 → 양식의 시각적 정합성 손상.

본 결함은 단순 web Canvas effect 누락이 아니라 **한컴 워터마크 통합 처리** 와 **AI 분야 활용을 위한 메타정보 제공** 까지 확장된다.

## 범위 (6 영역)

### 포함

#### 영역 1 — Web Canvas effect 적용

`src/renderer/web_canvas.rs::draw_image` 또는 별도 emit 경로에서 `image.effect` (GrayScale / BlackWhite / Pattern8x8) 를 Canvas 2D context filter 로 적용.

후보 구현:
- **방안 A**: `ctx.filter = 'grayscale(100%)'` 등 CSS filter 문자열 (간단, 단 SVG feColorMatrix 와 미세 차이 가능)
- **방안 B**: `ctx.drawImage` 후 ImageData 직접 조작 (정확, 느림)
- **방안 C**: 오프스크린 SVG 에 feColorMatrix 적용 후 Canvas 로 전사 (정확, 복잡)

선택 권장: **방안 A** (성능 우선, CLI SVG 와 시각 미세 차이는 작업지시자 시각 판정 게이트로 점검)

#### 영역 2 — Web Canvas brightness / contrast 적용

CSS filter 의 brightness/contrast 매핑:
- SVG feComponentTransfer: slope = (100 + contrast) / 100, intercept = (0.5 - 0.5×slope) + brightness/100
- CSS filter: `brightness(N%)` (N%=원본 → 100% 가 1.0) + `contrast(N%)` (100%=원본)

매핑 변환:
- HWP brightness=-50 → CSS `brightness(0.5)` (즉 0.5 = -50%)
- HWP contrast=70 → CSS `contrast(1.7)`
- 단 HWP 의 brightness 와 CSS brightness 는 정의가 다름 (HWP: linear intercept, CSS: scalar multiplier) → 정확한 매핑은 시각 판정으로 검증

선택 권장: **CSS filter 문자열 합성** + 시각 판정 게이트

#### 영역 3 — effect + brightness + contrast 합성 (워터마크 painter ordering)

CLI SVG 는 두 filter 를 중첩 `<g>` 로 적용 (먼저 grayscale → 다음 brightness/contrast). Canvas 는 단일 `ctx.filter` 문자열에 모두 합성:
```
ctx.filter = 'grayscale(100%) brightness(0.5) contrast(1.7)';
```

CSS filter 함수 순서가 시각 결과에 영향 → CLI SVG 의 painter 순서와 정합 확인 필요.

#### 영역 4 — PCX 배경 투명도 + 워터마크 결합

Task #514 의 PCX 변환은 흰색 (255,255,255) 픽셀을 투명 알파로 매핑한 RGBA PNG 출력. 워터마크 효과 적용 시:
- GrayScale + brightness(-50) + contrast(70) 후에도 알파 채널 보존
- CSS filter 가 RGBA 의 알파 채널을 침범하지 않는지 점검

검증: 복학원서.hwp 의 학교 로고 (PCX, 좌상단) + 엠블렘 (JPEG, 가운데) 모두 시각 판정.

#### 영역 5 — `rhwp dump` 에 image_attr 메타정보 추가

현재 dump 출력:
```
[1] 그림: bin_id=2, common=37128×37180 (131.0×131.2mm), orig=..., cur=..., tac=false
```

확장 후 (예상):
```
[1] 그림: bin_id=2, common=..., tac=false
[1]   [image_attr] effect=GrayScale brightness=-50 contrast=70 watermark=true
```

변경 위치: `src/main.rs` 의 4 개 dump 사이트 (line 1430 / 1644 / 1687 / 1763 / 1800).

#### 영역 6 — AI 메타정보 (PageLayerTree JSON 의 워터마크 식별)

`PaintOp::Image` JSON 에 `watermark: true/false` 필드 추가. 워터마크 판정 로직:
- 한컴 워터마크 프리셋 패턴 (`effect=GrayScale + brightness<0 + contrast>0` 또는 `+brightness>0 + contrast<0`) 매칭
- 또는 명시적 임계값 (`abs(brightness) >= 30 && abs(contrast) >= 30`)

정확한 판정 임계값은 한컴의 "워터마크 효과" 도구 동작 분석 후 결정 (Stage 1 진단 단계).

PR #510 의 schemaVersion 정책 정합 — additive change 이므로 schemaVersion 유지.
