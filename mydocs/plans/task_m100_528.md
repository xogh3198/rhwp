## 0. v1 → v2 변경 사유

[Stage 1 보고서](../working/task_m100_528_stage1.md) 측정으로 v1 가설 (Hangul Jamo Extended-A/B 폰트 fallback) 부정확이 확정됨.

실제 본질: `samples/exam_kor.hwp` p17 의 옛한글 음절은 **HanCom Private Use Area (PUA) 인코딩** (U+E000-F8FF + U+F0000+) 으로 저장. 기존 발견된 Issue #512 의 영역 정합.

## 1. 배경 / 증상

`samples/exam_kor.hwp` 페이지 17 의 중세국어 표기 미렌더:

```
'다'(되다)         ← 빈 자리 = PUA 옛한글 음절 글리프 부재
'(혼자)'
'​​'              ← Supplementary PUA-A (U+F0854, F0855)
'스스​'
```

p17 만 약 25 종 PUA 옛한글 음절 (Basic PUA 22 + Supplementary PUA-A 3).

## 2. 본질

### 2-1. HanCom 의 옛한글 인코딩 진화

| 한컴 버전 | 옛한글 인코딩 |
|----------|-----------|
| 한/글 2010 이전 | **PUA 영역 (U+E000-F8FF) + Supplementary PUA-A** — 한컴 자체 정의 매핑 |
| 한/글 2010+ | **KS X 1026-1:2007 자모 영역** (U+1100-11FF + U+A960-A97F + U+D7B0-D7FF) — 표준 |

`exam_kor.hwp` 는 한/글 2010+ 환경에서 작성됐으나 옛한글 입력은 **이전 판 호환 (PUA)** 으로 보존된 것으로 추정.

### 2-2. HncPUAConverter

한컴은 자체 변환 도구 ([hncpuaconverter.htm](https://github.com/edwardkim/rhwp/blob/devel/mydocs/manual/hwp/Help/extracted/hwpbase/hncpuaconverter.htm)):

```
HncPUAConverter.exe — 이전 판 한/글 PUA 옛한글 → KS X 1026-1:2007 유니코드 자모 시퀀스 변환
```

rhwp 는 본 변환을 자체 구현해야 정상 렌더링 가능.

### 2-3. 한컴 PDF 가 정상 렌더되는 이유

한컴 자체 폰트 (함초롬바탕 LVT 등) 가 PUA 코드포인트에 옛한글 음절 글리프를 직접 보유. 그래서 변환 없이도 PDF 출력 가능. rhwp 는 라이선스로 이 폰트 사용 불가 → PUA 변환 필수.

### 2-4. 코드 인프라 현황

긍정적:
- `src/renderer/composer.rs::convert_pua_enclosed_numbers` — 기존 PUA 변환 패턴 (테두리 숫자) 존재
- `src/renderer/style_resolver.rs::detect_lang_category` — 옛한글 영역 (U+1100-11FF, U+A960-A97F, U+D7B0-D7FF) 인식 코드 존재
- Task #122 — 옛한글 자모 클러스터 폭 계산 + 렌더링 인프라 완료

미구현:
- **PUA → KS X 1026-1 자모 시퀀스 변환 매핑 표**
- **Composer 단계 PUA 변환 호출**

## 3. 해결 방향

### 3-1. 변환 파이프라인

```
[원본 IR: PUA U+E38A 등]
       ↓ Composer 단계
[map_pua_old_hangul] — 매핑 표 룩업
       ↓
[KS X 1026-1 자모 시퀀스: U+1112 + U+1163 + U+11AB 등]
       ↓ 기존 Task #122 자모 클러스터 처리
[렌더 자모 클러스터 → 폰트 합자 (CCMP/LJMO/VJMO/TJMO)]
       ↓
[Source Han Serif K Old Hangul / Noto Serif KR 등 OFL 폰트로 합자 완성]
```

### 3-2. 매핑 표 확보

5개 후보 자료원 ([Stage 1 보고서 §4-2](../working/task_m100_528_stage1.md)):

1. **KS X 1026-1:2007 부속서** — 정식 표준 (유료)
2. **HncPUAConverter.exe 역공학** — Windows 환경 (라이선스 주의)
3. **함초롬바탕 LVT cmap** — 폰트 자체 매핑 (라이선스 주의)
4. **오픈소스 PUA 매핑 표** — `nuhwp` / `hwp.js` / `pyhwp` 등 점검
5. **gen-pua + 한컴 PDF 비교** — Task #509 패턴 정합

가장 라이선스 안전한 1차 시도: **(4) 오픈소스 영역 → (5) gen-pua 검증**.

### 3-3. 변환 적용 위치 — Composer

`src/renderer/composer.rs` — 렌더 직전 단일 진실 원천. Task #122 의 자모 클러스터 폭 계산과 정합. 기존 `convert_pua_enclosed_numbers` 패턴 정합.

### 3-4. 폰트 fallback (부차)

PUA 변환 후 결과는 KS X 1026-1 자모 영역. 이 영역 합자를 지원하는 폰트가 fallback 체인에 필요:
- **Noto Serif KR** — 자모 영역 cmap 보유, 합자 피처 (CCMP) 일부 지원 (Stage 4 검증)
- 미지원 시 **본명조 (Source Han Serif K)** 옛한글 subset 보강 (v1 영역 정합 — 부분 활용)
