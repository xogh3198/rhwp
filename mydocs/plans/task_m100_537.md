## 1. 문제 정의

`samples/21_언어_기출_편집가능본.hwp` 의 다수 위치에서 첫 답안(① 직후 ②) 또는 박스 직후 첫 문단의 줄/문단 간격이 IR(`LINE_SEG.vpos`) 보다 정확히 1 ls(=716 HU = 9.55 px) 좁게 렌더링된다.

작업지시자가 명시한 위치 11곳 모두 동일 패턴:
- TAC `<보기>` 표 직후 첫 답안 문단 (3/6/9/12/15/17/18/23/24/27/29 번 ①→②)
- 박스(셀) 내 첫 단락 직후 두 번째 단락 (7p 르포르 박스, 9p 수피즘 박스)

## 2. 근본 원인 (사전 분석에서 확정)

세 메커니즘의 상호작용으로 drift 716 HU 가 영구화됨:

1. **`prev_tac_seg_applied` 가드** (`src/renderer/layout.rs:1434`): TAC 표 직후 paragraph 의 vpos 보정을 건너뜀.
2. **trailing line_spacing 제외** (`src/renderer/layout/paragraph_layout.rs:2645-2654`, Task #479): paragraph 마지막 줄에서 `line_spacing` 가산 생략 → pi(첫 답)의 advance 가 IR delta 보다 1 ls 짧음.
3. **`lazy_base` drift 동결** (`layout.rs:1497-1507`): pi=39 부터 vpos 보정 재개되지만 lazy_base 를 sequential y 에서 역산하므로 716 HU drift 가 base=716 으로 그대로 고정됨.

결과: 첫 답(①)만 IR vpos 정확, 후속 답(②~⑤)은 IR vpos − 716 HU 위치로 일괄 시프트 → ①→② gap 만 좁아 보임.

## 3. 수정 방향 (3안 비교)

| 안 | 내용 | 영향 범위 | 회귀 위험 | 채택 |
|-----|------|-----------|-----------|------|
| **A** | lazy_base 산출 시 첫 paragraph 의 IR vpos 와 sequential y 가 일치하지 않으면 base=0 으로 강제 (또는 IR vpos 직접 사용) | 매우 국소: `layout.rs:1497-1507` | 낮음 | **유력 1순위** |
| B | TAC 표 직후 paragraph 도 vpos 보정 허용 (`prev_tac_seg_applied` 게이트 완화) | 중간: TAC 표 후처리 (layout.rs:2536-2554) 와 이중 보정 가능성 | 중간 (TAC 분할/다중 TAC 영향) | 2순위 (A 실패 시) |
| C | trailing-ls 정책을 `lh_sum + n*ls` 로 통일 | 광범위: typeset/pagination/layout/cell 전체 | **높음** (Task #452→#479 전환 이력 역행) | 보류 |

본 task 는 **A안** 으로 진행.

### A안 상세

`layout.rs:1497-1507` 의 lazy_base 역산 로직을, **현재 paragraph 의 first_vpos 가 신뢰 가능하면(>0 이고 prev_pi 의 first_vpos 보다 큼) 직접 anchor 로 사용** 하도록 변경. 이렇게 하면 pi=39 처리 시 base 가 IR 절대좌표 기준으로 산출되어 drift 가 즉시 해소된다.

근사 구현 스케치 (실 구현은 Stage 2 에서 결정):
```rust
let lazy_base = if let (Some(curr_first), Some(prev_first)) = (
    paragraphs.get(item_para).and_then(|p| p.line_segs.first()).map(|s| s.vertical_pos),
    paragraphs.get(prev_pi).and_then(|p| p.line_segs.first()).map(|s| s.vertical_pos),
) {
    // 현 vpos 가 유효하면 prev 의 first_vpos 를 base 로 사용 (sequential drift 무시)
    if curr_first > prev_first { prev_first }
    else { prev_vpos_end - y_delta_hu }   // 기존 fallback
} else {
    prev_vpos_end - y_delta_hu
};
```

단, prev_pi 가 column 첫 항목이면 base = 0 이 자연스러움. 정확한 식은 Stage 2 에서 검증.
