## 1. 문제 정의

`samples/21_언어_기출_편집가능본.hwp` 의 두 그룹에서 한컴 PDF 대비 줄간격이 좁게 렌더링됨.

### 그룹 A — 지문 시작 표시 직후 (9곳)
"[X~Y] 다음 글을 읽고 물음에 답하시오." 와 다음 문단 사이.
2p[4~6], 4p[7~9], 5p[10~12], 7p[13~15], 8p[16~18], 10p[19~21], 11p[22~24], 13p[25~27], 14p[28~30].

### 그룹 B — 글박스(InFrontOfText Shape) 안 본문 paragraph 사이 (2곳)
- 7p 박스 안 "르포르는 1789년..." 과 윗 문단 사이
- 9p 박스 안 "더불어 수피즘..." 과 윗 문단 사이

## 2. 사전 진단 결과

### 2.1 그룹 A 정량 (페이지 2 [4~6] 대표)

| paragraph | text | vpos | ps_id | line | lh | ls |
|-----------|------|------|-------|------|----|----|
| pi=44 | "[4~6] 다음 글을..." | 0 | 10 | 165% | 1100 | 716 |
| pi=45 | (빈 문단) | 1816 | **35** | **60%** | 1100 | **−440** |
| pi=46 | "15세기 초..." | 2476 | 21 | 165% | 1100 | 716 |

- rhwp 측정 gap pi=44→pi=46 = 33.01 px = **IR 정확**
- 의심: 빈 paragraph + 60% 음수 ls 처리. 한컴은 빈 paragraph 의 visual height 를 lh 그대로 (또는 다른 방식으로) 처리할 가능성.

### 2.2 그룹 B 정량 (페이지 9 "더불어 수피즘")

페이지 9 col 0 의 PartialParagraph pi=181 (lines 8..13) → pi=182 "더불어 수피즘..." gap.

| paragraph | text | vpos | ls |
|-----------|------|------|----|
| pi=181 line 12 | (윗 문장) | 7264 (col-reset) | 716 |
| pi=182 line 0 | "더불어 수피즘..." | 9080 | 716 |

- IR delta = 1816 HU = **24.21 px** 기대
- SVG 측정 gap = **14.67 px** (= 1100 HU, 1 lh) → 9.55 px(=1 ls) 좁음
- Task #537 의 같은 본질 패턴이지만 fix 가 적용 안 됨 — vpos correction 의 page_path 사용 (col 첫 항목이 PartialParagraph) 으로 lazy_base 보정 미발동.

## 3. 수정 방향 (3안 비교)

### A안: 그룹 B 만 우선 처리 (page_path 도 trailing-ls 보정)
`layout.rs:1518-1527` 의 page_path 식 `end_y = col_anchor_y + (vpos_end - base) * scale` 가 base = vpos_page_base = col 첫 항목 first vpos. 정확. 그런데 sequential y_offset 이 trailing-ls drift 로 IR 보다 부족 → end_y > y_offset → applied 적용되어야 함.

페이지 9 의 경우 applied=true 인데도 여전히 좁음? VPOS_CORR debug 출력으로 확인 필요 (Stage 1).

### B안: 그룹 A 처리 (60% line spacing 빈 paragraph 의 height 보정)
빈 paragraph (cc==0 or text.is_empty()) 인 경우 line_spacing 이 음수여도 advance 를 lh × 100% 로 강제. paragraph_layout.rs 의 ComposedLine 줄 advance 에서 처리.

영향 범위 광범위 — 모든 빈 paragraph 의 advance 변경.

### C안: 그룹 A 의 한컴 정확 동작 확인 후 결정
한컴 2010/2020 PDF 와 비교하여 실제 advance 측정. 음수 ls 의 한컴 동작이 명세 룰인지 휴리스틱인지 식별 후 룰화.

## 4. 채택안

**A안 우선 (그룹 B)** + **그룹 A 는 사전 조사 후 결정**.

이유:
- 그룹 B 는 Task #537 의 본질과 동일 (716 HU drift), fix 만 page_path 에 확장.
- 그룹 A 는 빈 paragraph + 음수 ls 라는 별개 본질이며, 한컴 동작 확인 없이 변경 시 광범위 회귀 위험. 메모리 룰 ("룰과 휴리스틱 구분") 에 따라 한컴 명세 자문 필요.
