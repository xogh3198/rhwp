# Task #370 v2 보충 보고서

## 추가 변경 배경

v1 변경(`Noto Serif CJK KR` 추가) 후, **macOS Chrome** 환경 검증 결과 여전히 볼드 표시 안 됨.

## 원인

macOS의 폴백 매칭 흐름:

1. 함초롬바탕 — 미설치(한컴 미설치 환경) → 다음
2. Batang / 바탕 — macOS 미설치 → 다음
3. **AppleMyungjo — macOS 기본 설치 → 매칭 종료**

`AppleMyungjo`는 Regular variant만 보유. macOS Chrome은 CJK 글리프에 대해 font-weight 합성을 잘 수행하지 않음(알려진 동작). 이번 v1에서 추가한 `Noto Serif CJK KR`은 AppleMyungjo 뒤에 있어 도달 불가.

## v2 변경 (Option A)

`'AppleMyungjo'` **앞에** `'Nanum Myeongjo'` 삽입.

- Nanum Myeongjo는 macOS 10.9+ 기본 설치이며 **실제 Bold variant 보유**.
- AppleMyungjo보다 우선 매칭 → macOS Chrome에서 진짜 Bold 매칭 성공.

| 위치 | 신규 체인 |
|------|----------|
| `mod.rs:560` (한글 세리프) | `'Batang','바탕','Nanum Myeongjo','AppleMyungjo','Noto Serif KR','Noto Serif CJK KR',serif` |
| `mod.rs:567` (영문 세리프) | (동일) |

## 검증

- `cargo test`: **1055 passed, 0 failed**
- 골든 스냅샷 3건 폰트 체인 갱신.

## 영향 범위 (재검토)

- macOS: 한글 세리프 기본 face가 AppleMyungjo → Nanum Myeongjo로 변경. 두 폰트 모두 한글 명조 계열이므로 사용자 지정 폰트(함초롬바탕 등)가 미설치인 fallback 상황에서만 발생하는 변화이며, Bold variant 매칭이라는 핵심 효과를 위해 수용 가능한 trade-off.
- Windows / Linux: Batang(Win) 또는 Nanum Myeongjo(설치된 경우)가 매칭. Nanum Myeongjo는 Linux에서도 `nanum-myeongjo` 패키지 설치 환경에서 매칭됨. 미설치 환경은 기존과 동일하게 `Noto Serif CJK KR`로 떨어짐.
