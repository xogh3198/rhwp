## 영향

### 1. "PDF 와 일치" 검증의 무효화

이전까지 v0.7.x 사이클의 다수 task 에서 "PDF 와 일치" 를 검증 기준 사용:
- Task #347 (Q27/Q28 미세 오프셋)
- Task #356 (페이지 분기 오버플로 — PR #360)
- Task #359 (typeset fit drift — k-water-rfp)
- Task #361 (page_num 갱신)
- Task #362 (kps-ai PartialTable + Square wrap)

이 task 들의 검증이 **누구의 한컴 PDF** 인지 명시되지 않은 경우, 다른 환경에서 회귀로 보일 수 있음.

### 2. 외부 컨트리뷰터 PR 의 검증 신뢰도

PR #251 (Visual Diff 발견 3건 수정), PR #360 (페이지 분기) 등 외부 컨트리뷰터 PR 이 "한컴 PDF 와 일치" 를 검증 기준으로 사용. 다음 이슈 발생 가능:

- 컨트리뷰터 환경에서 PDF 일치 → 작업지시자 환경 재검증 시 불일치
- 머지 후 작업지시자 환경에서 회귀로 보임
- 양쪽 모두 정상이라고 주장하지만 결과 다름

### 3. 이슈 #253 (한컴 PDF 기준 Visual Diff 파이프라인)

이슈 #253 의 핵심 가정 — **"한컴 PDF = 단일 권위 기준"** 이 본 정황으로 무효. 파이프라인 구축 보류 권장.

## 처리 원칙 (적용 의무)

### 1. PDF 는 정답지가 아니다

**rhwp 검증 기준에서 "PDF 와 일치" 를 1차 기준으로 수용 금지**.

- 결정적 검증 우선: `cargo test --lib` (회귀 0), `svg_snapshot` (rhwp 자체 일관성)
- 시각 검증은 **어느 환경의 한컴 PDF** 인지 명시 의무

### 2. PR 검토 시

- 컨트리뷰터의 "PDF 와 일치" 검증 결과는 **참고 자료**. 1차 기준 아님
- 작업지시자 환경 재검증 필수 (특히 페이지 분할 영향 task)
- PR 댓글에서 명시: "PDF 검증은 결정적이지 않으니 시각 + cargo test 회귀 0 으로 판정"

### 3. Task 검증 시

- "PDF 와 일치" 사용 시 **commit / 보고서 / CHANGELOG 에 환경 명시** (예: "한컴2024, macOS Sequoia, 함초롬바탕 설치 환경")
- 가능하면 다중 환경 시각 점검
- 결정적 검증 (cargo test, svg_snapshot) 을 1차로

### 4. 외부 컨트리뷰터 안내

`CONTRIBUTING.md` / `README.md` / PR 템플릿에 명시:
- "PDF 와 일치" 만으로 PR 통과 보장 안 됨
- 검증 기준: cargo test + 시각 결과 + 작업지시자 환경 재현

### 5. 이슈 #253 보류

PDF 가 정답지로서 신뢰성이 입증되기 전까지 Visual Diff 파이프라인 구축 보류.
재개 조건:
- 환경별 PDF 차이 정량 측정 완료
- 차이 패턴 분석 (어느 환경이 권위? 어떤 측면이 일치?)
- 단일 권위 기준 결정 (또는 결정 불가 시 다른 검증 전략)
