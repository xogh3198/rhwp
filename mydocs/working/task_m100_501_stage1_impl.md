### 핵심 회귀 본질

**rs=2 셀 안의 r=1 (위쪽 정원) / r=2 (아래쪽 현원) 분배 — 27.45 / 7.99 비균등** (정상 17.07 / 17.07).

작업지시자 표현 \"윗쪽 행 너무 적게 계산\" 의 원인 추적:
- 합계 셀 (rs=2 cs=2) 의 두 행 분배에서 정상 균등 (1/2 = 17.07) 이 아닌 **27.45 / 7.99** 분배
- r=1 정원 영역이 27.45px → 정원 라벨 + 숫자 영역 너무 큼
- r=2 현원 영역이 7.99px → 현원 라벨 + 숫자 영역 너무 작음 → **현원 텍스트가 위 영역 침범**
- 작업지시자 시각 = 합계 라벨 위치/현원 위치 비대칭 = \"분배 잘못\" / \"위쪽 너무 적음\" (인덱스 행 r=0 도 포함)

### 회귀 origin 확정 — 셀[21] (현 원, r=2) 의 큰 padding

dump 출력:
```
셀[21] r=2,c=2 rs=1,cs=1 h=1280 w=4935 pad=(141,141,1700,1700) aim=false bf=16 paras=1 text=\"현 원\"
```

- 셀[21] 의 IR padding: top=1700 HU, bottom=1700 HU (정상 141 HU 의 **12배**)
- `resolve_cell_padding` (table_layout.rs:775-781): aim=false 일 때 `(cell.padding > table.padding)` 이면 cell 우선 → pad_top + pad_bottom = **45.33px**
- content_height (\"현 원\" 1줄, corrected_line_height) ≈ 21px
- **required_height = 21 + 45.33 = ~66px** ★
- 1단계 row_heights[2] = max(17.07, 66) = **66px**
- raw_table_height = 26.4 (r=0) + 17.07 (r=1) + **66** (r=2 셀[21] 큰 pad) + 17.07 (r=3) + ... ≈ 매우 큼 (예 ≈ **257px**)
- common.h = 146.13 → **scale ≈ 0.568** → 모든 행 비례 축소
- r=2 = 66 × 0.568 = **37.5px** → 그러나 SVG 측정 r=2 = 7.99 (불일치)

### Scale 계산 재추적

위 시나리오는 SVG 측정과 정확히 정합 안 됨. 다른 영역 (예: 표 안의 다른 셀 페이지 분배 또는 paragraph_layout 의 두 번째 별도 path) 이 r=2 영역을 7.99 로 더 축소.

→ Stage 2 정정 시 본 정합 추가 정밀화 필요. 다만 회귀 origin 영역 (큰 cell padding 1700 HU + scale) 은 명확.
